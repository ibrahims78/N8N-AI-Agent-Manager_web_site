NEW PACKAGE: @workspace/n8n-nodes-catalog

UPDATER SCRIPT (scripts/fetch.mjs)
  - Calls GET /repos/ibrahims78/n8n/git/trees/master?recursive=1
  - Filters paths matching: packages/nodes-base/nodes/**/*.node.json
  - Fetches each .node.json from raw.githubusercontent.com (with concurrency 8)
  - Aggregates 430 records into lib/n8n-nodes-catalog/data/catalog.json
  - Run with:   pnpm --filter @workspace/n8n-nodes-catalog run fetch

DATABASE SCHEMA: lib/db/src/schema/node_catalog.ts
  Tables created (drizzle-kit push):

  node_catalog
  ------------
    id                 serial PK
    node_type          text UNIQUE        e.g. "n8n-nodes-base.slack"
    display_name       text
    folder             text                e.g. "Slack"
    file_name          text
    is_trigger         boolean
    node_version       text
    codex_version      text
    categories         jsonb                ["Communication", "HITL"]
    subcategories      jsonb                {"HITL": ["Human in the Loop"]}
    aliases            jsonb                ["human","approval","wait",...]
    primary_docs_url   text
    credential_docs_url text
    examples           jsonb                [{label,url,icon}, ...]
    icon_url           text
    raw_json           jsonb                full original .node.json
    created_at / updated_at  timestamps
  Indexes: node_type (unique), display_name, is_trigger

  node_catalog_meta
  -----------------
    id                 serial PK
    source_url         text
    branch             text
    total_nodes        integer
    fetched_at         timestamp
    created_at         timestamp
  Records each refresh operation for observability.

BACKEND SERVICE: artifacts/api-server/src/services/nodeCatalog.service.ts
  Public API:
    seedNodeCatalog()           Upsert all 430 nodes from static catalog.json
    listCatalog(opts)           Paginated list with search / category / trigger
    getCategoriesWithCounts()   Returns categories sorted by count
    lookupNode(query)           Resolves by node-type, folder, or alias (exact)
    searchCatalogDb(q, limit)   Fuzzy search by displayName/aliases/categories
    refreshCatalogFromGitHub()  Live fetches latest data from GitHub
    getCatalogStatus()          Returns total + last fetched_at metadata

REST ENDPOINTS: artifacts/api-server/src/routes/catalog.routes.ts
  All endpoints require JWT auth. Mount: /api/catalog

  GET  /api/catalog/status                Total + last fetched info
  GET  /api/catalog/categories            All categories with counts
  GET  /api/catalog                       List with filters:

                                            ?category=Communication

                                            ?limit=24&offset=0

================================================================================
  PHASE 2 — AI AGENT ENHANCEMENT
================================================================================

NEW TOOL: lookup_node_catalog
  Added to: artifacts/api-server/src/services/agentTools.ts
  Schema:
    {
      name: "lookup_node_catalog",
      params: { query: string }
    }
  Behaviour:
    1. First tries exact match via lookupNode() (covers node type, folder,
       fileName, and 256 aliases such as "human approval" → Slack).
    2. Falls back to fuzzy search across display name, aliases, categories,
       returning up to 10 matches.
    3. Returns: nodeType, displayName, isTrigger, categories, subcategories,
       aliases, primaryDocsUrl, credentialDocsUrl, examples (up to 5).

  Why it matters
  --------------
  Users in Arabic frequently use unusual phrasings like "أريد عقدة بشرية"
  ("I want a human node") which previously had to be guessed by the LLM.
  The catalog's alias index now resolves these directly to the correct n8n
  node type, and the LLM can also surface the official docs URL to the user
  without hallucinating links.

================================================================================
  HOW TO USE
================================================================================
  1. Browse the catalog:    Sign in → click "Nodes Catalog" in the left nav.
  2. Search/filter:         Type "slack", "human approval", or pick a category.
  3. Open docs:             Click "Docs" or "Auth" on any node card.
  4. Refresh from GitHub:   Settings page → "n8n Nodes Catalog" → "Refresh Now".
  5. Update the static seed file (advanced):
        pnpm --filter @workspace/n8n-nodes-catalog run fetch

  AI Agent usage:
  When chatting with the assistant, ask things like:
    "Which n8n node should I use for human-in-the-loop approvals?"
    "Give me the official docs link for the Postgres node."
    "List all trigger nodes in the Communication category."
  The agent will call lookup_node_catalog automatically.
