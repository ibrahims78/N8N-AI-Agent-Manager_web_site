ملخص
-----
تم بنجاح دمج 430 عقدة رسمية من n8n داخل مدير وكيل N8N الذكي.
كل عقدة تحمل: الاسم المعروض، الفئات، الفئات الفرعية، المرادفات (مصطلحات
بحث بديلة)، علامة Trigger، رابط التوثيق الرسمي، رابط توثيق بيانات
الاعتماد، وروابط أمثلة سير العمل من مدوّنة n8n.io.

سكربت التحديث (scripts/fetch.mjs)

  - يُرشّح المسارات المطابقة لـ: packages/nodes-base/nodes/**/*.node.json
  - يجلب كل ملف ‎.node.json من raw.githubusercontent.com (تزامن 8)

  - طريقة التشغيل:
      pnpm --filter @workspace/n8n-nodes-catalog run fetch

مخطط قاعدة البيانات: lib/db/src/schema/node_catalog.ts
  الجداول التي تم إنشاؤها (drizzle-kit push):

  node_catalog
  ------------
    id                  serial PK
    node_type           text UNIQUE        مثل: "n8n-nodes-base.slack"
    display_name        text
    folder              text                مثل: "Slack"
    file_name           text
    is_trigger          boolean
    node_version        text
    codex_version       text
    categories          jsonb               ["Communication", "HITL"]
    subcategories       jsonb               {"HITL": ["Human in the Loop"]}
    aliases             jsonb               ["human","approval","wait", ...]
    primary_docs_url    text
    credential_docs_url text
    examples            jsonb               [{label, url, icon}, ...]
    icon_url            text
    raw_json            jsonb               الملف الأصلي ‎.node.json كاملًا
    created_at / updated_at  timestamps
  الفهارس: node_type (فريد)، display_name، is_trigger

  node_catalog_meta
  -----------------
    id           serial PK
    source_url   text
    branch       text
    total_nodes  integer
    fetched_at   timestamp
    created_at   timestamp
  يُسجّل كل عملية تحديث لأغراض الرصد والمتابعة.

خدمة الواجهة الخلفية: artifacts/api-server/src/services/nodeCatalog.service.ts
  الواجهة العامة:
    seedNodeCatalog()           إدخال/تحديث 430 عقدة من ملف catalog.json
    listCatalog(opts)           قائمة مع تقسيم صفحات/بحث/فئة/Trigger
    getCategoriesWithCounts()   إرجاع الفئات مرتبة حسب العدد
    lookupNode(query)           بحث مباشر عن طريق node-type/folder/alias
    searchCatalogDb(q, limit)   بحث تقريبي على الاسم/المرادفات/الفئات
    refreshCatalogFromGitHub()  جلب مباشر لأحدث البيانات من GitHub
    getCatalogStatus()          إرجاع الإجمالي وآخر تاريخ تحديث

نقاط REST: artifacts/api-server/src/routes/catalog.routes.ts
  جميع المسارات تتطلب مصادقة JWT. يتم تركيبها على: /api/catalog

  GET  /api/catalog/status           الإجمالي وآخر تحديث
  GET  /api/catalog/categories       جميع الفئات مع الأعداد
  GET  /api/catalog                  قائمة مع فلاتر:
                                       ?search=xxx
                                       ?category=Communication
                                       ?trigger=true|false
                                       ?limit=24&offset=0
  GET  /api/catalog/lookup/:query    بحث مباشر بالنوع/المجلد/المرادف
  POST /api/catalog/refresh   [أدمن] تحديث مباشر من GitHub

دمج البذر: artifacts/api-server/src/seed.ts
  يُبذَر الكتالوج تلقائيًا عند كل تشغيل للخادم (إدخال/تحديث آمن للتكرار).
  أول بذر في هذه الجلسة أدخل 430 عقدة، والمرات اللاحقة ستحدّث المتغير فقط.

أداة جديدة: lookup_node_catalog
  أُضيفت في: artifacts/api-server/src/services/agentTools.ts
  المخطط:
    {
      name: "lookup_node_catalog",
      params: { query: string }
    }
  السلوك:
    1. يحاول أولًا المطابقة الدقيقة عبر lookupNode() (تُغطي نوع العقدة،
       والمجلد، واسم الملف، و256 مرادفًا مثل "human approval" → Slack).
    2. عند الفشل ينتقل إلى البحث التقريبي على الاسم المعروض والمرادفات
       والفئات، ويُرجع حتى 10 نتائج.
    3. يُرجع: nodeType، displayName، isTrigger، categories، subcategories،
       aliases، primaryDocsUrl، credentialDocsUrl، examples (حتى 5).

  أهمية ذلك
  ----------
  المستخدمون باللغة العربية يستخدمون كثيرًا صياغات غير مألوفة مثل
  "أريد عقدة بشرية" التي كان النموذج اللغوي مضطرًا لتخمينها سابقًا.
  فهرس المرادفات في الكتالوج يحلّ هذه العبارات مباشرة إلى نوع العقدة
  الصحيح في n8n، ويمكن للنموذج أيضًا تقديم رابط التوثيق الرسمي للمستخدم
  دون اختلاق روابط.

  طريقة الاستخدام
================================================================================
  1. تصفّح الكتالوج:    سجّل الدخول → اضغط "كتالوج العقد" في القائمة الجانبية.
  2. البحث/الفلترة:     اكتب "slack" أو "human approval" أو اختر فئة.
  3. فتح التوثيق:       اضغط "التوثيق" أو "المصادقة" على أي بطاقة.
  4. تحديث من GitHub:   صفحة الإعدادات → "كتالوج عُقد n8n" → "تحديث الآن".
  5. تحديث ملف البذر الثابت (متقدّم):
        pnpm --filter @workspace/n8n-nodes-catalog run fetch

  استخدام الوكيل الذكي:
  أثناء المحادثة مع المساعد، اسأل أسئلة مثل:
    "ما عقدة n8n التي يجب أن أستخدمها لموافقات الإنسان في السير (HITL)؟"
    "أعطني رابط التوثيق الرسمي لعقدة Postgres."
    "اعرض جميع عُقد Trigger في فئة التواصل."
  سيقوم الوكيل باستدعاء lookup_node_catalog تلقائيًا.
