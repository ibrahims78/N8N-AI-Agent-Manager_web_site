## المرحلة 1: عرض Markdown وتلوين الكود

### الهدف
استبدال عرض النص العادي بعرض Markdown كامل مع تلوين الكود، وذلك لكل رسائل الوكيل (assistant).

### المشكلة الحالية
في `MessageContent` (السطر 186)، يعرض الكود النص كـ `whitespace-pre-wrap` فقط، ويتعامل مع JSON فقط دون بقية عناصر Markdown كالعناوين والقوائم والجداول.

### الحزم المطلوبة
```bash
pnpm --filter @workspace/n8n-manager add react-markdown remark-gfm react-syntax-highlighter
pnpm --filter @workspace/n8n-manager add -D @types/react-syntax-highlighter
```

### خطوات التطبيق

#### الخطوة 1.1 — إنشاء مكوّن MarkdownRenderer
إنشاء ملف جديد:
`artifacts/n8n-manager/src/components/MarkdownRenderer.tsx`

المحتوى:
- استيراد `ReactMarkdown` من `react-markdown`
- استيراد `remarkGfm` من `remark-gfm`
- استيراد `SyntaxHighlighter` من `react-syntax-highlighter`
- استيراد ثيم (مثل `oneDark` أو `oneLight`) من `react-syntax-highlighter/dist/esm/styles/prism`
- تعريف `components` لكل عنصر Markdown:
  - `code`: إذا كان inline → `<code>` بتنسيق مضمّن، إذا كان block → `SyntaxHighlighter` مع زر نسخ
  - `h1, h2, h3`: عناوين بأحجام مناسبة
  - `ul, ol, li`: قوائم منسّقة
  - `blockquote`: بتنسيق مميز (border-s + bg-muted)
  - `table, thead, tbody, tr, th, td`: جداول منسّقة
  - `a`: روابط تفتح في نافذة جديدة مع أيقونة ExternalLink
  - `p`: فقرات بـ `leading-relaxed mb-2`
  - `hr`: فاصل بـ border

## المرحلة 2: تكبير/تصغير النافذة وطيّ الـ Sidebar

### الهدف
منح المستخدم تحكماً كاملاً في مساحة المحادثة: طيّ الـ Sidebar، سحب حدوده، وتكبير قسم المحادثة للملء الكامل.

### المشكلة الحالية
الـ Sidebar ثابت بعرض `w-72` (288px)، ولا توجد أي إمكانية لتغيير الحجم أو الطيّ.

### خطوات التطبيق

#### الخطوة 2.1 — حالة الطيّ (Collapse)
في `ChatPage`:
- إضافة `useState<boolean>` باسم `sidebarCollapsed` (القيمة الافتراضية: `false`)
- عند `sidebarCollapsed === true`: الـ Sidebar يصبح `w-12` ويعرض فقط أيقونات بدون نص
- زر طيّ في رأس الـ Sidebar (أيقونة `PanelLeftClose` / `PanelLeftOpen` من lucide)
- انتقال سلس باستخدام `transition-all duration-300`

#### الخطوة 2.2 — شريط السحب (Resize Handle)
بين الـ Sidebar وقسم المحادثة:
- إضافة `div` بعرض `4px` وـ cursor `col-resize`
- استخدام `useRef` لتتبع عرض الـ Sidebar الحالي
- استخدام `onMouseDown` + `document.addEventListener('mousemove')` + `document.addEventListener('mouseup')` لتنفيذ السحب
- تقييد العرض بين حد أدنى (`180px`) وأقصى (`480px`)
- حفظ العرض في `localStorage` بمفتاح `chat_sidebar_width`
- استرجاع العرض المحفوظ عند تحميل الصفحة من `localStorage`

#### الخطوة 2.3 — وضع التكبير الكامل (Full-screen)
- إضافة `useState<boolean>` باسم `fullscreen`
- زر تكبير في شريط الأدوات (أيقونة `Maximize2` / `Minimize2`)
- عند `fullscreen === true`: إخفاء الـ Sidebar بالكامل ووضع قسم المحادثة على `fixed inset-0 z-50`
- مفتاح `Escape` للخروج من التكبير

#### الخطوة 2.4 — التخطيط عند الطيّ
عند `sidebarCollapsed`:
- عرض فقط أيقونة كل محادثة (أو الحرف الأول من اسمها)
- عرض أيقونة `+` فقط بدلاً من زر "محادثة جديدة" الكامل
- tooltip يظهر عنوان المحادثة عند التمرير

## المرحلة 3: رفع الملفات والصور

### الهدف
السماح للمستخدم برفع صور (لعرض أخطاء أو مخططات) وملفات (JSON أو YAML أو TXT) لمساعدة الوكيل على فهم السياق بشكل أفضل.

### القرار المعماري
الملفات والصور تُرفق مع الرسالة النصية كـ base64 أو كنص مستخرج، وتُرسل للـ API ضمن نفس الطلب الحالي (لا حاجة لـ endpoint رفع منفصل في البداية).

### خطوات التطبيق

#### الخطوة 3.1 — واجهة رفع الملفات في Frontend

##### إضافة زر المرفقات
في شريط الإدخال (السطر 710-731 في chat.tsx):
- إضافة زر 📎 (أيقونة `Paperclip` من lucide) على يسار الـ textarea
- عند النقر: تفعيل `<input type="file" hidden>` باستخدام `useRef`
- الأنواع المقبولة: `image/*,application/json,text/plain,text/yaml,application/x-yaml`
- الحد الأقصى للحجم: 5MB

##### معاينة المرفقات
إضافة قسم فوق الـ textarea (قبل زر الإرسال) يعرض:
- **للصور**: صورة مصغرة 48px x 48px مع زر ❌ للحذف
- **للملفات النصية**: أيقونة الملف + اسمه + حجمه مع زر ❌
- ألوان مختلفة: JSON (أزرق)، YAML (بنفسجي)، TXT (رمادي)، صور (أخضر)

##### Drag & Drop
- إضافة `onDragOver`, `onDragLeave`, `onDrop` لقسم المحادثة
- عند السحب: تغيير خلفية قسم المحادثة لـ `bg-accent/5 border-2 border-dashed border-accent`
- عند الإفلات: معالجة الملف كما في اختيار الملف العادي

#### الخطوة 3.2 — معالجة الملفات

##### للصور
```typescript
const reader = new FileReader();
reader.readAsDataURL(file);
reader.onload = () => {
  setAttachments(prev => [...prev, {
    type: 'image',
    name: file.name,
    data: reader.result as string, // base64 data URL
    size: file.size,
  }]);
};
```

##### للملفات النصية (JSON, YAML, TXT)
```typescript
const reader = new FileReader();
reader.readAsText(file);
reader.onload = () => {
  setAttachments(prev => [...prev, {
    type: 'file',
    name: file.name,
    content: reader.result as string, // النص الكامل
    size: file.size,
    mimeType: file.type,
  }]);
};
```

#### الخطوة 3.3 — إرسال المرفقات مع الرسالة

في `handleSend`:
- بناء رسالة موسّعة تدمج النص مع محتوى الملفات:
```typescript
let fullContent = text;
for (const att of attachments) {
  if (att.type === 'file') {
    fullContent += `\n\n[ملف مرفق: ${att.name}]\n\`\`\`\n${att.content}\n\`\`\``;
  } else if (att.type === 'image') {
    fullContent += `\n\n[صورة مرفقة: ${att.name}]`;
    // الصورة ترسل منفصلة في body إن دعم الـ API ذلك، وإلا تُذكر فقط
  }
}
```
- مسح المرفقات بعد الإرسال: `setAttachments([])`

#### الخطوة 3.4 — عرض المرفقات في الرسائل
في فقاعة رسالة المستخدم:
- إذا كانت الرسالة تحتوي على صورة مرفقة → عرض الصورة فوق النص (من base64)
- إذا كانت تحتوي على ملف → عرض شارة بيانية (اسم الملف + أيقونة)

#### الخطوة 3.5 — Backend (اختياري للمرحلة الأولى)
في الوضع الأولي، لا تغيير في الـ Backend — المحتوى يُدمج في النص العادي.
للمرحلة المستقبلية:
- إضافة حقل `attachments` في جدول `messages` بقاعدة البيانات
- إضافة endpoint رفع منفصل إذا تجاوز حجم الصور 1MB

### الاختبار
- [ ] النقر على زر المرفقات يفتح نافذة اختيار الملفات
- [ ] اختيار صورة يعرض معاينة مصغرة
- [ ] اختيار JSON يعرض شارة بيانية مع الاسم والحجم
- [ ] زر ❌ على المرفق يحذفه من القائمة
- [ ] سحب صورة وإفلاتها في نافذة المحادثة يضيفها
- [ ] الملف يُرسل مع الرسالة ومحتواه يظهر في رد الوكيل
- [ ] رفع ملف أكبر من 5MB يعرض رسالة خطأ
- [ ] الأنواع غير المدعومة (PDF) تُرفض مع رسالة واضحة

## المرحلة 7: تنظيم وتحسين الـ Sidebar

### الهدف
جعل الـ Sidebar أكثر فائدة مع بحث وتسمية وتثبيت.

### خطوات التطبيق

#### الخطوة 7.1 — بحث في المحادثات
في رأس الـ Sidebar تحت زر "محادثة جديدة":
- إضافة `input` بحث صغير مع أيقونة `Search`
- تصفية `conversations` بناءً على نص البحث (يبحث في `title`)
- مسح البحث بزر ❌ عند وجود نص
- لا يحتاج API call — التصفية في frontend فقط

#### الخطوة 7.2 — تسمية المحادثة
- النقر المزدوج على عنوان المحادثة يحوّله لـ `input` قابل للتعديل
- عند الضغط على Enter أو الخروج: إرسال `PUT /api/chat/conversations/{id}` لتحديث العنوان
- عند الخروج بدون تغيير: الإلغاء

#### الخطوة 7.3 — تثبيت المحادثات
- زر 📌 (أيقونة `Pin`) يظهر عند التمرير
- حفظ قائمة المحادثات المثبتة في `localStorage`
- عرض المثبتة في قسم خاص "مثبتة" في أعلى الـ Sidebar قبل المجموعات الزمنية
- زر 📌 مختلف الحالة عند التثبيت (ملون)

#### الخطوة 7.4 — معلومات إضافية في كل محادثة
- إضافة عدد الرسائل بجانب اسم المحادثة: `{messageCount} رسالة`
- مؤشر نوع المحادثة: دائرة صغيرة (أخضر لـ "query"، أزرق للـ "generation")
- تنسيق التاريخ: عرض "منذ X دقيقة/ساعة/يوم" بدلاً من التاريخ الكامل

### الاختبار
