## 1) الهدف من التحديث

رفع جودة صفحة الأدلة لتصبح:
- **احترافية بصرياً** بترويسة وإحصاءات وتنسيق Markdown أنيق.
- **شاملة في عرض المعلومات** (الفئة، الطول، تاريخ التحديث، حالة الترجمة، حالة التحرير اليدوي، رابط المصدر).
- **قوية في البحث**: بحث نصّي شامل عبر محتوى كل الأدلة (server-side) مع ترتيب حسب عدد التطابقات وإبراز مقاطع مختصرة.
- **داعمة للترجمة الاحترافية**: إضافة طبقة «تحرير يدوي» (Manual Override) تُحفظ منفصلة عن النسخة المُترجَمة آلياً، فلا يُمحى عمل المسؤول عند إعادة الترجمة.
- **موثوقة في الجلب والمزامنة**: عرض حالة كل دليل (متوفر / غير متوفر / تحرير يدوي / به خطأ) مع تقدّم لحظي للجلب الجماعي.

## 2) ملخّص التغييرات

### قاعدة البيانات
أعمدة جديدة في `guides_docs`:

| العمود | النوع | الغرض |
|---|---|---|
| `manual_override_markdown` | `text` | نصّ التحرير اليدوي (يفوق النسخة الآلية في العرض). |
| `manual_override_at` | `timestamptz` | تاريخ آخر تحرير يدوي. |
| `manual_override_by` | `integer` | معرّف المستخدم الذي حرّر. |
| `manual_override_note` | `text` | ملاحظة اختيارية تشرح التغيير. |

> ✱ تصميم الـ override **منفصل** عن `markdown` (نسخة المصدر)، فلا يُمحى عند إعادة الجلب أو إعادة الترجمة.

### نقاط النهاية الجديدة

| Method | Path | الصلاحية | الوظيفة |
|---|---|---|---|
| `GET` | `/api/catalog/docs-advanced/guides/stats` | مستخدم | إحصاءات شاملة (إجمالي، EN، AR، Overrides، آخر تحديث). |
| `GET` | `/api/catalog/docs-advanced/guides/search?q=&lang=` | مستخدم | بحث نصّي شامل في محتوى الأدلة مع snippets. |
| `PUT` | `/api/catalog/docs-advanced/guides/:slug/manual?lang=` | أدمن | حفظ تحرير يدوي للدليل (يُستخدم خصوصاً لتلميع الترجمة العربية). |
| `DELETE` | `/api/catalog/docs-advanced/guides/:slug/manual?lang=` | أدمن | مسح التحرير اليدوي والعودة للنسخة الآلية. |

### تعديلات على الموجود
- `GET /guides`: يعيد الآن لكل دليل: `hasMarkdown`, `hasOverride`, `length`, `error`, `updatedAt` — بترتيب ثابت (فئة، ثم عنوان).
- `GET /guides/:slug`: أصبح يضمّن `effectiveMarkdown` (override أو الأصلي)، ولا يعيد الجلب فوق التحرير اليدوي إلا بطلب صريح (`force=true`).
- `fetchGuide` / `fetchArabicGuide`: لم تتغيّر سلوكياً لكنها لم تعد تمسّ عمود الـ override أبداً.

## 3) صفحة الواجهة — التصميم الجديد

### أ) الترويسة الاحترافية
- أيقونة مع عنوان وصفّ قصير يوضّح ماهية الأدلة.
- شريط أدوات للأدمن: «جلب الكل (EN)» و«جلب + ترجمة AR».
- **5 بطاقات إحصاء**: الإجمالي، التغطية الإنجليزية (مع نسبة %), التغطية العربية (مع نسبة %), عدد التحريرات اليدوية، آخر تحديث.
- شريط بحث كبير بـ debounce 250ms يبحث في **محتوى** الأدلة (لا فقط العناوين).

### ب) شريط تقدّم لحظي
عند تشغيل الجلب الجماعي يظهر شريط ملوّن يعرض:
- المرحلة (جلب EN / ترجمة AR) مع أيقونة.
- اسم الدليل الحالي.
- العدّاد `done/total` + شريط تقدّم نسبي.

### جـ) القائمة الجانبية الذكية
- مجمَّعة حسب الفئة مع شارة عدد لكل فئة بلون مميَّز:
  - Glossary (أزرق) · Workflows (بنفسجي) · Expressions (كهرماني) · Credentials (أخضر) · Hosting (وردي) · API (سماوي).
- لكل دليل أيقونات حالة:
  - ✏️ كهرماني = يوجد تحرير يدوي.
  - ⚠️ أحمر = غير متوفر.
- عند البحث: تتحوّل القائمة إلى **نتائج البحث** مع snippets ودرجة التطابق وعدد المرات.

### د) لوحة المحتوى
**الترويسة العلوية** تعرض:
- شارة الفئة بلون مميَّز.
- شارة «تحرير يدوي» (إن وجد) أو شارة «خطأ» (إن وجد).
- العنوان كاملاً.
- شريط ميتاداتا: تاريخ آخر تحديث، عدد الأحرف، رابط المصدر.
- شريط أزرار الإجراءات: نسخ، تنزيل `.md`، إعادة الجلب، تحرير، مسح التحرير.

**جسم الصفحة** يعرض المحتوى بـ Markdown احترافي:
- عناوين بحدود سفلية وفصل واضح بين المستويات.
- روابط بلون التمييز مع underline عند المرور.
- جداول بحدود ورؤوس مظلَّلة.
- اقتباسات بشريط جانبي ملوّن وخلفية خفيفة.
- أكواد بخلفية وحدود وحجم مناسب.
- صور بحدود وزوايا منحنية.
- في العربية: line-height 1.85 وحجم خط أكبر (15px) لراحة القراءة.

### هـ) وضع التحرير اليدوي (للأدمن)
عند الضغط على ✏️:
- ينقسم العرض إلى **عمودَين** (Markdown على جانب، معاينة حيّة على الآخر).
- زرّا «حفظ» و«إلغاء» في الترويسة.
- بعد الحفظ: تُحدَّث القائمة والـ stats والمستند معاً.
- عرض شريط معلوماتي تحت العنوان: «نسخة محرَّرة يدوياً — تجاوز للترجمة الآلية. آخر تعديل: …».

### و) حالات فارغة وتفاعلات احترافية
- شاشة ترحيب أنيقة قبل اختيار دليل.
- رسالة واضحة عند عدم وجود نتائج بحث.
- زرّ «محاولة الجلب» يظهر تلقائياً عند فشل تحميل دليل (للأدمن).
- جميع الأزرار تحوي `title` (tooltip) باللغة الفعّالة.

## 4) كيف يعمل البحث؟

البحث **server-side** (`searchGuides`):
1. يجلب كل أدلة اللغة المختارة من DB.
2. لكل دليل: يستخدم `effectiveMarkdown` (override إن وجد، وإلّا النسخة الآلية).
3. يحسب عدد التطابقات (case-insensitive) ويُرجِع snippet حول أوّل تطابق (±80 / +200 حرف).
4. يُعطي تطابق العنوان وزناً إضافياً (+5).
5. يرتّب النتائج حسب عدد التطابقات (تنازلياً) ويقتصر على 25 نتيجة.

العميل يستدعي البحث بعد debounce 250ms ويتجاهل الاستعلامات الأقصر من حرفَين.

## 5) دورة حياة الترجمة العربية

```
┌──────────────┐    fetchAllGuides       ┌──────────────────┐
│  GitHub Raw  │───────EN markdown──────▶│ guides_docs (en) │
└──────────────┘                          └────────┬─────────┘
                                                   │
                                                   ▼ translateMarkdownToArabic
                                          ┌──────────────────┐
                                          │ guides_docs (ar) │
                                          │   markdown       │ ← الترجمة الآلية
                                          │   override       │ ← التحرير اليدوي (يفوز)
                                          └────────┬─────────┘
                                                   │
                                                   ▼ getGuide
                                       effectiveMarkdown =
                                       override ?? markdown
```

- **إعادة الترجمة لا تمسّ override** — حتى لو ضُغط «جلب + ترجمة AR» مع `force=true`.
- لمسح override يدوياً: استخدم زرّ 🗑️ في الترويسة (للأدمن).

## 6) الجلب الكامل والصحيح

- مصدر البيانات: مستودع `n8n-io/n8n-docs` على GitHub raw.
- لكل دليل في `CORE_GUIDE_PAGES` (17 دليلاً حالياً) جرّب الخدمة المسارَين:
  - `docs/<path>.md`
  - `docs/<path>/index.md`
- المحتوى يُنظَّف من بقايا MkDocs (`{{ … }}`, `[[ … ]]`) قبل الحفظ.
- يُحفظ `sourceSha` (SHA1) للكشف عن التغييرات في المزامنة الدورية.
- نسخة احتياطية على القرص في `lib/n8n-nodes-catalog/guides/<lang>/<slug>.md` (للنجاة من إعادة بناء DB).
- **التحرير اليدوي** يُكتب أيضاً على القرص.

التقدّم اللحظي يأتي عبر **SSE** من `POST /guides/refresh-all` بحدثَين:
- `progress` — { phase, done, total, current }
- `done` — { fetched, failed, translated, translateFailed }

## 8) كيف تستخدم الصفحة (للمسؤول)

1. **أوّل مرّة**: اضغط «جلب + ترجمة AR» — سيجلب الـ 17 دليلاً ويترجمها (يستغرق دقائق حسب مزوّد الذكاء).
2. **مراجعة الترجمة**: اختر دليلاً، اضغط أيقونة ✏️، عدِّل النصّ، استعرض المعاينة الحيّة، ثم احفظ.
3. **تحديث دوري**: بعد فترة، اضغط «جلب الكل (EN)» لتحديث المصدر — ترجمتك المحفوظة لن تتأثّر.
4. **العودة للترجمة الآلية**: اضغط 🗑️ بجانب أزرار التحرير — ستظهر النسخة الآلية فوراً.
5. **البحث**: اكتب أيّ كلمة (≥ حرفَين) في شريط البحث؛ ستُعرض النتائج بمقاطع وعدد التطابقات.

## 10) اختبار سريع (Manual QA)

1. سجّل دخول كأدمن (`admin / 123456`).
2. اذهب إلى صفحة الأدلة → تحقّق من ظهور البطاقات (الإجمالي، EN، AR، Overrides، آخر تحديث).
3. اضغط «جلب الكل (EN)» → تأكّد من ظهور شريط التقدّم وزيادة عدّاد EN.
4. اختر «Glossary» → يظهر بتنسيق احترافي.
5. اضغط ✏️ → عدّل سطراً → احفظ → تحقّق من ظهور شارة «تحرير يدوي» وأنّ التعديل ظاهر.
6. اضغط «إعادة الجلب» (🔄) → التعديل اليدوي يبقى.
7. اضغط 🗑️ → التعديل يُمسح وتعود النسخة الأصلية.
8. اكتب كلمة في البحث (مثل `node`) → تظهر نتائج مرتّبة مع snippets.
9. بدِّل اللغة → القائمة والمحتوى يعيدان التحميل تلقائياً.
