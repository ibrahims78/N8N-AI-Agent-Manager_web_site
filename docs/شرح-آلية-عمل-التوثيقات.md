> آخر تحديث: 22 أبريل 2026 — يغطي جميع التحسينات العشرة المُطبَّقة.

## 2) جدول قاعدة البيانات بالتفصيل

### `node_docs` — المستند الكامل لكل عقدة/لغة
| العمود | الغرض |
|---------|--------|
| `nodeType`, `language` | المفتاح المركّب |
| `markdown`, `sourceUrl`, `etag`, `fetchedAt` | الإصدار من المصدر |
| `manual_override_markdown` | نص محرَّر يدوياً يعلو على المصدر |
| `manual_override_at`, `manual_override_by` | معلومات التحرير |
| `term_vector` (JSONB) | مفردات BM25 الجاهزة |

### `node_docs_history`
كل تعديل (سواء جلب جديد أو تحرير يدوي أو استرجاع) يُحفظ كصف هنا، فيمكن لمدير التطبيق فتح "السجل" واستعراض جميع الإصدارات والرجوع لأي منها.

### `node_doc_sections`
يقسّم كل مستند إلى أقسام (`## ...`) ويحفظ نصها + قاموس مفردات. هذا هو المحرك الفعلي للبحث الشامل.

### `guides_docs`
جدول مستقل للأدلة العامة الـ17 (Glossary، Workflow Logic، Triggers، …) بنفس آلية التخزين والترجمة.

### `docs_sync_settings`
صف وحيد يتحكّم في:
- `auto_sync_enabled`
- `interval_hours`
- `last_run_at`, `next_run_at`
- خيارات الترجمة التلقائية بعد الجلب.

## 3) خدمة `nodeDocs.service.ts` (الأصلية، تم توسيعها)

- `getEnglishDoc(nodeType, force)` — يُعيد أولاً **التحرير اليدوي إن وجد**، وإلا يستخدم cache، وإلا يجلب من GitHub. بعد الجلب يحلّ شيفرات `--8<--` (تضمينات MkDocs) بشكل صامت.
- `getArabicDoc(nodeType, force)` — يضيف نفس منطق التحرير اليدوي ثم cache ثم ترجمة عبر OpenAI proxy مع قاموس مصطلحات (TRANSLATE_GLOSSARY) لضمان اتساق المفاهيم العربية.
- يدعم `force=true` لإعادة الجلب وتجاوز cache.

عند ترجمة طويلة، يُقسَّم النص إلى chunks لا تتجاوز سعة النموذج، ثم يُجمَّع الناتج. التوقيت العادي ≈ 60 ثانية لكل مستند، وهذا قابل للمعالجة في الخلفية ضمن "لوحة تحكم التوثيق (مدير)".

## 4) خدمة `docsAdvanced.service.ts` (الجديدة)

تجمع كل المنطق الجديد:

1. **Reindex Sections** — `reindexAllSections()`: يقطّع كل مستند إلى أقسام ويبني شعاع مفردات BM25 لكل قسم. تم فهرسة **3,647** قسماً.
2. **Global Search** — `searchAllDocs(query, lang)`: يطبّق ترتيب BM25 على المفردات المخزّنة + يُعيد مقتطفات (snippets) حول كلمات البحث.
3. **History & Rollback** — `listHistory()`, `rollbackTo(historyId)`.
4. **Manual Override CRUD** — `setManualOverride()`, `clearManualOverride()`.
5. **Auto-Sync** — `runScheduledSync()` يُستدعى من scheduler؛ يطلب `HEAD` بـ ETag ويتجاوز ما لم يتغيّر.
6. **Operations Extraction** — يُسلّم لـ `extractOperations()` في `docsHelpers.ts` (تم تثبيت كاشف القوائم النقطية ليلتقط 40 عملية في عقدة Slack مثلاً).
7. **Guides Fetching** — `fetchAllGuides()` يجلب 17 دليلاً عاماً مرة واحدة (يعمل حالياً في الخلفية وقت كتابة هذا المستند).
8. **Export** — `buildExportHtml(markdown, lang)` يبني HTML منسّق، RTL تلقائي، CSS مدمج، وزر طباعة يحوّل لـ PDF داخل المتصفح.

## 5) واجهة المستخدم — التكامل في `nodes-catalog.tsx`

تم حقن المكوّنات الجديدة في الصفحة دون كسر التصميم القائم:

```
صفحة كتالوج العقد
 ├── لوحة تحكم التوثيق (مدير)
 │    └── SyncSettingsCard           ← المحور 6
 ├── GlobalDocsSearch (بطاقة منفصلة) ← المحور 3
 ├── شريط الفلاتر القديم (لم يُمسّ)
 └── DocsViewer (شريط الأدوات)
      ├── OperationsPanel            ← المحور 7
      ├── HistoryPanel               ← المحور 4
      ├── WorkflowJsonViewer         ← المحور 10
      ├── ManualEditor (مدير فقط)   ← المحور 5
      └── زر إعادة الجلب القديم
```

كذلك أُضيف بند تنقّل جديد **"أدلة n8n"** يفتح `pages/guides.tsx` لتصفّح الـ17 دليلاً عاماً. مكوّن `DocResp` تم توسيعه ليحمل `manualOverride: boolean` فيستطيع المحرّر إظهار شارة "نسخة محرَّرة" مقابل "نسخة المصدر".

## 6) كيف يصل المستخدم إلى كل ميزة

| الميزة | المسار في الواجهة |
|---|---|
| بحث BM25 شامل | كتالوج العقد ← شريط البحث العلوي ← اضغط نتيجة لتفتح العقدة |
| سجل وإصدارات | اعرض مستند عقدة ← زر "السجل" ← استرجاع أي إصدار (مدير) |
| تحرير يدوي | اعرض مستند عقدة ← زر "تحرير" (مدير) → يحفظ كـ override |
| استخراج العمليات | داخل عرض العقدة ← زر "العمليات" |
| عارض JSON | داخل عرض العقدة ← زر "Workflow JSON" — يكتشف كتل ` ```json ` ويعرضها كشجرة قابلة للطيّ |
| مزامنة تلقائية | لوحة تحكم التوثيق (مدير) ← بطاقة "المزامنة التلقائية" |

## 7) ملاحظات تشغيلية مهمة

- **الذكاء الاصطناعي**: نستخدم بروكسي Replit OpenAI (`gpt-5-mini`). دالة embeddings غير مدعومة على هذا البروكسي ولا على Gemini، لذا اعتمدنا BM25 بدل التشابه الدلالي — وهي مناسبة جداً لطبيعة وثائق n8n التقنية.
- **الـ ETag**: يقلّل تكلفة الاستعلامات ويمنع كتابة سجلات تاريخ زائفة عند عدم تغيّر المصدر.
- **الترجمة الكسولة**: لا نترجم 541 مستنداً دفعة واحدة — تتم الترجمة عند أول طلب عربي للعقدة، ويمكن لمشغّل اللوحة فرض ترجمة شاملة في الخلفية.
- **التحرير اليدوي يبقى محفوظاً عبر المزامنة**: حتى لو غيّر n8n وثيقة Slack، يبقى نصّك المحرّر هو المعروض، مع إمكانية المقارنة عبر السجل.
- **PDF**: تخلّينا عن puppeteer وتركنا للمستخدم تحويل الصفحة المنسّقة عبر طباعة المتصفح، مما يحلّ مشاكل الخطوط العربية ويُبقي الحجم منخفضاً.
