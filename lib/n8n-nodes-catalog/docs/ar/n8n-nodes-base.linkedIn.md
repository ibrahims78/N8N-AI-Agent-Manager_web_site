# عقدة LinkedIn

استخدم عقدة LinkedIn لأتمتة العمل في LinkedIn، ودمج LinkedIn مع تطبيقات أخرى. تدعم n8n إنشاء المنشورات.

في هذه الصفحة، ستجد قائمة بالعمليات التي تدعمها عقدة LinkedIn وروابط لموارد إضافية.

> **بيانات الاعتماد**
>
> ارجع إلى [بيانات اعتماد LinkedIn](/integrations/builtin/credentials/linkedin.md) للحصول على إرشادات حول إعداد المصادقة.

> **يمكن استخدام هذه العقدة كأداة ذكاء اصطناعي**
>
> يمكن استخدام هذه العقدة لتعزيز قدرات وكيل الذكاء الاصطناعي. عند استخدامها بهذه الطريقة، يمكن تعيين العديد من المعاملات تلقائيًا، أو بمعلومات موجهة بواسطة الذكاء الاصطناعي - اكتشف المزيد في [توثيق معاملات أداة الذكاء الاصطناعي](/advanced-ai/examples/using-the-fromai-function.md).

    * إنشاء

## المعاملات

* **Post As**: اختر ما إذا كنت تريد النشر كـ **شخص** أو **مؤسسة**.
* **Person Name or ID** و **Organization URN**: أدخل معرفًا للشخص أو المؤسسة.

	/// note | النشر كمؤسسة
	إذا كنت تنشر كمؤسسة، أدخل رقم المؤسسة في حقل URN. على سبيل المثال، `03262013` وليس `urn:li:company:03262013`.
	///
	
* **Text**: محتويات المنشور.
* **Media Category**: استخدم هذا عند تضمين صور أو عناوين URL للمقالات في منشورك.

## الموارد ذات الصلة

ارجع إلى [توثيق API الخاص بـ LinkedIn](https://learn.microsoft.com/en-us/linkedin/) لمزيد من المعلومات حول الخدمة.

## ماذا تفعل إذا كانت عمليتك غير مدعومة

إذا كانت هذه العقدة لا تدعم العملية التي تريد القيام بها، يمكنك استخدام [عقدة طلب HTTP](/integrations/builtin/core-nodes/n8n-nodes-base.httprequest/index.md) لاستدعاء API الخاص بالخدمة.

يمكنك استخدام بيانات الاعتماد التي أنشأتها لهذه الخدمة في عقدة طلب HTTP:

1. في عقدة طلب HTTP، حدد **Authentication** > **Predefined Credential Type**.
2. حدد الخدمة التي تريد الاتصال بها.
3. حدد بيانات الاعتماد الخاصة بك.

ارجع إلى [عمليات API المخصصة](/integrations/custom-operations.md) لمزيد من المعلومات.
