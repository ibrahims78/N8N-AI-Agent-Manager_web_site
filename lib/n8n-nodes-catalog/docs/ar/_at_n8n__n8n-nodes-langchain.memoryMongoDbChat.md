# عقدة MongoDB Chat Memory

استخدم عقدة MongoDB Chat Memory لاستخدام MongoDB كخادم [ذاكرة](/glossary.md#ai-memory) لتخزين سجل الدردشة.

في هذه الصفحة، ستجد قائمة بالعمليات التي تدعمها عقدة MongoDB Chat Memory، وروابط لموارد إضافية.

> **بيانات الاعتماد**
>
> يمكنك العثور على معلومات المصادقة لهذه العقدة [هنا](/integrations/builtin/credentials/mongodb.md).

> **حل المعاملات في العقد الفرعية**
>
> تتصرف العقد الفرعية بشكل مختلف عن العقد الأخرى عند معالجة عناصر متعددة باستخدام تعبير.
>
> تأخذ معظم العقد، بما في ذلك العقد الجذرية، أي عدد من العناصر كمدخلات، وتعالج هذه العناصر، وتخرج النتائج. يمكنك استخدام التعبيرات للإشارة إلى عناصر المدخلات، وتقوم العقدة بحل التعبير لكل عنصر على حدة. على سبيل المثال، بالنظر إلى مدخلات من خمس قيم `name`، يتم حل التعبير `` لكل اسم على حدة.
>
> في العقد الفرعية، يتم حل التعبير دائمًا للعنصر الأول. على سبيل المثال، بالنظر إلى مدخلات من خمس قيم `name`، يتم حل التعبير `` دائمًا للاسم الأول.

## معاملات العقدة

*   **Session Key**: أدخل المفتاح المراد استخدامه لتخزين الذاكرة في بيانات سير العمل.
*   **Collection Name**: أدخل اسم المجموعة لتخزين سجل الدردشة فيها. سيقوم النظام بإنشاء المجموعة إذا لم تكن موجودة.
*   **Database Name**: أدخل اسم قاعدة البيانات لتخزين سجل الدردشة فيها. إذا لم يتم توفيره، فسيتم استخدام قاعدة البيانات من بيانات الاعتماد.
*   **Context Window Length**: أدخل عدد التفاعلات السابقة التي يجب أخذها في الاعتبار للسياق.
