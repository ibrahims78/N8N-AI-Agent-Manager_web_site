# عقدة Zendesk

استخدم عقدة Zendesk لأتمتة العمل في Zendesk، ودمج Zendesk مع تطبيقات أخرى. تدعم n8n مجموعة واسعة من ميزات Zendesk بشكل مدمج، بما في ذلك إنشاء وحذف التذاكر والمستخدمين والمؤسسات.

ستجد في هذه الصفحة قائمة بالعمليات التي تدعمها عقدة Zendesk وروابط لموارد إضافية.

> **بيانات الاعتماد**
>
> ارجع إلى [بيانات اعتماد Zendesk](/integrations/builtin/credentials/zendesk.md) للحصول على إرشادات حول إعداد المصادقة.

> **يمكن استخدام هذه العقدة كأداة للذكاء الاصطناعي**
>
> يمكن استخدام هذه العقدة لتعزيز قدرات وكيل الذكاء الاصطناعي. عند استخدامها بهذه الطريقة، يمكن تعيين العديد من المعاملات تلقائيًا، أو باستخدام معلومات موجهة بواسطة الذكاء الاصطناعي - اكتشف المزيد في [توثيق معاملات أداة الذكاء الاصطناعي](/advanced-ai/examples/using-the-fromai-function.md).

## العمليات

*   Ticket (التذكرة)
    *   إنشاء تذكرة
    *   حذف تذكرة
    *   جلب تذكرة
    *   جلب جميع التذاكر
    *   استعادة تذكرة معلقة
    *   تحديث تذكرة
*   Ticket Field (حقل التذكرة)
    *   جلب حقل تذكرة
    *   جلب جميع حقول التذاكر النظامية والمخصصة
*   User (المستخدم)
    *   إنشاء مستخدم
    *   حذف مستخدم
    *   جلب مستخدم
    *   جلب جميع المستخدمين
    *   جلب مؤسسات المستخدم
    *   جلب البيانات المتعلقة بالمستخدم
    *   البحث عن المستخدمين
    *   تحديث مستخدم
*   Organization (المؤسسة)
    *   إنشاء مؤسسة
    *   حذف مؤسسة
    *   عد المؤسسات
    *   جلب مؤسسة
    *   جلب جميع المؤسسات
    *   جلب البيانات المتعلقة بالمؤسسة
    *   تحديث مؤسسة

> **سلوك استبدال الوسوم**
>
> عند استخدام عملية "تحديث تذكرة" في عقدة Zendesk وتحديد حقل `Tag Names or IDs`، سيتم **استبدال** القائمة الكاملة للوسوم (tags) الموجودة على التذكرة. سيتم إزالة أي وسوم غير مدرجة في التحديث من التذكرة بسبب كيفية معالجة Zendesk API لتحديثات الوسوم افتراضيًا.
>
> **لتجنب الإزالة العرضية للوسوم:**
>
> -   أولاً، استرجع وسوم التذكرة وادمجها مع وسومك الجديدة قبل التحديث.
> -   بدلاً من ذلك، استخدم عقدة HTTP Request مع خاصية `additional_tags` الخاصة بـ Zendesk لإضافة وسوم دون إزالة الوسوم الموجودة.
> -   يمكنك أيضًا استدعاء نقطة النهاية `/tags` الخاصة بالتذكرة لإضافة وسوم دون استبدال الوسوم الموجودة ([توثيق نقطة نهاية وسوم Zendesk](https://developer.zendesk.com/api-reference/ticketing/ticket-management/tags/)).
>
> راجع التوثيق الرسمي للحصول على التفاصيل: [إضافة وسوم إلى التذاكر دون الكتابة فوق الوسوم الموجودة](https://developer.zendesk.com/documentation/ticketing/managing-tickets/adding-tags-to-tickets-without-overwriting-existing-tags/).

## ماذا تفعل إذا كانت عمليتك غير مدعومة

إذا كانت هذه العقدة لا تدعم العملية التي ترغب في القيام بها، يمكنك استخدام [عقدة HTTP Request](/integrations/builtin/core-nodes/n8n-nodes-base.httprequest/index.md) لاستدعاء API الخاص بالخدمة.

يمكنك استخدام بيانات الاعتماد التي أنشأتها لهذه الخدمة في عقدة HTTP Request:

1.  في عقدة HTTP Request، حدد **المصادقة** > **نوع بيانات الاعتماد المُحددة مسبقًا**.
2.  حدد الخدمة التي ترغب في الاتصال بها.
3.  حدد بيانات الاعتماد الخاصة بك.

ارجع إلى [عمليات API المخصصة](/integrations/custom-operations.md) لمزيد من المعلومات.
