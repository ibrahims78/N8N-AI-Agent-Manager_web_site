# عقدة أداة HTTP Request

> **إصدار الأداة القديم**
>
> تستخدم النسخ الجديدة من عقدة أداة HTTP Request التي تضيفها إلى سير العمل العقدة القياسية [HTTP Request](/integrations/builtin/core-nodes/n8n-nodes-base.httprequest/index.md) كأداة. تصف هذه الصفحة عقدة أداة HTTP Request القديمة والمستقلة.
>
> يمكنك تحديد إصدار الأداة المستخدم في سير العمل الخاص بك عن طريق التحقق مما إذا كانت العقدة تحتوي على خاصية **Add option** عند فتح العقدة على اللوحة. إذا كان هذا الزر موجودًا، فأنت تستخدم الإصدار الجديد، وليس الإصدار الموصوف في هذه الصفحة.

تعمل أداة HTTP Request تمامًا مثل عقدة [HTTP Request](/integrations/builtin/core-nodes/n8n-nodes-base.httprequest/index.md)، ولكنها مصممة للاستخدام مع [وكيل ذكاء اصطناعي](/glossary.md#ai-agent) كأداة لجمع المعلومات من موقع ويب أو API.

في هذه الصفحة، ستجد قائمة بالعمليات التي تدعمها عقدة HTTP Request وروابط لموارد إضافية.

> **بيانات الاعتماد**
>
> ارجع إلى [بيانات اعتماد HTTP Request](/integrations/builtin/credentials/httprequest.md) للحصول على إرشادات حول إعداد المصادقة.

> **حل المعاملات في العقد الفرعية**
>
> تتصرف العقد الفرعية بشكل مختلف عن العقد الأخرى عند معالجة عناصر متعددة باستخدام تعبير.
>
> تأخذ معظم العقد، بما في ذلك العقد الجذرية، أي عدد من العناصر كمدخلات، وتعالج هذه العناصر، وتخرج النتائج. يمكنك استخدام التعبيرات للإشارة إلى عناصر المدخلات، وتقوم العقدة بحل التعبير لكل عنصر على حدة. على سبيل المثال، بالنظر إلى مدخلات من خمس قيم `name`، يتم حل التعبير `` لكل اسم على حدة.
>
> في العقد الفرعية، يتم حل التعبير دائمًا إلى العنصر الأول. على سبيل المثال، بالنظر إلى مدخلات من خمس قيم `name`، يتم حل التعبير `` دائمًا إلى الاسم الأول.
