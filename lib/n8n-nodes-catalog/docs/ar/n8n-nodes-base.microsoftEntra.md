# عقدة Microsoft Entra ID

استخدم عقدة Microsoft Entra ID لأتمتة سير العمل في Microsoft Entra ID وتكامل Microsoft Entra ID مع التطبيقات الأخرى. يدعم n8n بشكل مدمج مجموعة واسعة من ميزات Microsoft Entra ID، والتي تشمل إنشاء المستخدمين والمجموعات واستردادها وتحديثها وحذفها، بالإضافة إلى إضافة المستخدمين إلى المجموعات وإزالتهم منها.

في هذه الصفحة، ستجد قائمة بالعمليات التي تدعمها عقدة Microsoft Entra ID، وروابط لموارد إضافية.

> **بيانات الاعتماد**
>
> يمكنك العثور على معلومات المصادقة لهذه العقدة [هنا](/integrations/builtin/credentials/microsoftentra.md).

> **دعم السحابة الحكومية**
>
> إذا كنت تستخدم مستأجر سحابة حكومية (حكومة الولايات المتحدة، وزارة الدفاع الأمريكية، أو الصين)، فتأكد من تحديد **Microsoft Graph API Base URL** المناسب في تهيئة بيانات اعتماد Microsoft Entra ID الخاصة بك.

## العمليات

*   **المجموعة**
    *   **إنشاء**: إنشاء مجموعة جديدة
    *   **حذف**: حذف مجموعة موجودة
    *   **استرداد**: استرداد بيانات لمجموعة محددة
    *   **استرداد متعدد**: استرداد قائمة بالمجموعات
    *   **تحديث**: تحديث مجموعة
*   **المستخدم**
    *   **إنشاء**: إنشاء مستخدم جديد
    *   **حذف**: حذف مستخدم موجود
    *   **استرداد**: استرداد بيانات لمستخدم محدد
    *   **استرداد متعدد**: استرداد قائمة بالمستخدمين
    *   **تحديث**: تحديث مستخدم
    *   **إضافة إلى مجموعة**: إضافة مستخدم إلى مجموعة
    *   **إزالة من مجموعة**: إزالة مستخدم من مجموعة

## ماذا تفعل إذا كانت عمليتك غير مدعومة

إذا كانت هذه العقدة لا تدعم العملية التي ترغب في القيام بها، يمكنك استخدام [عقدة طلب HTTP](/integrations/builtin/core-nodes/n8n-nodes-base.httprequest/index.md) لاستدعاء واجهة برمجة تطبيقات الخدمة.

يمكنك استخدام بيانات الاعتماد التي أنشأتها لهذه الخدمة في عقدة طلب HTTP:

1.  في عقدة طلب HTTP، حدد **المصادقة** > **نوع بيانات الاعتماد المحدد مسبقًا**.
2.  حدد الخدمة التي ترغب في الاتصال بها.
3.  حدد بيانات الاعتماد الخاصة بك.

ارجع إلى [عمليات API المخصصة](/integrations/custom-operations.md) لمزيد من المعلومات.

## المشكلات الشائعة

فيما يلي بعض الأخطاء والمشكلات الشائعة مع عقدة Microsoft Entra ID وخطوات حلها أو استكشاف أخطائها وإصلاحها.

### فشل تحديث خيارات السماح للمرسلين الخارجيين والاشتراك التلقائي للأعضاء الجدد

لا يمكنك تحديث خياري **Allow External Senders** و **Auto Subscribe New Members** مباشرة بعد إنشاء مجموعة جديدة. يجب عليك الانتظار بعد إنشاء مجموعة قبل أن تتمكن من تغيير قيم هذه الخيارات.

عند تصميم سير العمل التي تستخدم عقد Microsoft Entra ID متعددة لإنشاء المجموعات أولاً ثم تحديث هذه الخيارات، أضف [عقدة انتظار](/integrations/builtin/core-nodes/n8n-nodes-base.wait.md) بين العمليتين. تسمح عقدة الانتظار المهيأة للتوقف لمدة ثانيتين على الأقل بوقت لتهيئة المجموعة بالكامل. بعد الانتظار، يمكن لعملية التحديث أن تكتمل دون حدوث خطأ.
