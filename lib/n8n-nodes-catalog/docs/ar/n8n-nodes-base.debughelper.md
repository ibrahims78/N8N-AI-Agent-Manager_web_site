# Debug Helper

استخدم عقدة Debug Helper لإطلاق أنواع مختلفة من الأخطاء أو لإنشاء مجموعات بيانات عشوائية للمساعدة في اختبار سير عمل n8n.

## العمليات

حدد العملية باختيار **الفئة**:

*   **لا تفعل شيئًا**: لا تفعل أي شيء.
*   [**إطلاق خطأ**](#throw-error): إطلاق خطأ بالنوع والرسالة المحددين.
*   [**نفاد الذاكرة**](#out-of-memory): إنشاء حجم ذاكرة محدد لمحاكاة نفاد الذاكرة.
*   [**إنشاء بيانات عشوائية**](#generate-random-data): إنشاء بعض البيانات العشوائية بتنسيق محدد.

## معاملات العقدة

تعتمد معاملات العقدة على **الفئة** المختارة. لا تحتوي فئة **لا تفعل شيئًا** على أي معاملات أخرى.

### إطلاق خطأ

*   **Error Type**: حدد نوع الخطأ المراد إطلاقه. اختر من:
    *   **NodeApiError**
    *   **NodeOperationError**
    *   **Error**
*   **Error Message**: أدخل رسالة الخطأ المراد إطلاقها.

### نفاد الذاكرة

تضيف فئة نفاد الذاكرة معاملًا واحدًا، وهو **Memory Size to Generate**. أدخل الكمية التقريبية من الذاكرة المراد إنشاؤها.

### إنشاء بيانات عشوائية

*   **Data Type**: اختر نوع البيانات العشوائية التي ترغب في إنشائها. تتضمن الخيارات:
    *   **Address** (العنوان)
    *   **Coordinates** (الإحداثيات)
    *   **Credit Card** (بطاقة الائتمان)
    *   **Email** (البريد الإلكتروني)
    *   **IPv4**
    *   **IPv6**
    *   **MAC**
    *   **Nanoids**: إذا اخترت نوع البيانات هذا، فستحتاج أيضًا إلى إدخال:
        *   **Nanoid Alphabet**: الأبجدية التي سيستخدمها المُنشئ لإنشاء الـ nanoids.
        *   **Nanoid Length**: طول كل nanoid.
    *   **URL**
    *   **User Data** (بيانات المستخدم)
    *   **UUID**
    *   **Version** (الإصدار)
*   **Seed**: إذا كنت ترغب في إنشاء البيانات باستخدام بذرة محددة، فأدخلها هنا. يضمن ذلك إنشاء البيانات بشكل متسق. إذا كنت تفضل استخدام إنشاء البيانات العشوائية، فاترك هذا الحقل فارغًا.
*   **Number of Items to Generate**: أدخل عدد العناصر العشوائية التي ترغب في إنشائها.
*   **Output as Single Array**: ما إذا كان سيتم إنشاء البيانات كمصفوفة واحدة (مفعل) أو عناصر متعددة (معطل).
