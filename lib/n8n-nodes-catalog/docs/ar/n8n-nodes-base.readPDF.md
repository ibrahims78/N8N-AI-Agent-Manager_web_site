# استخراج من ملف

من الأنماط الشائعة في سير عمل n8n هو استقبال ملف، إما من [عقدة طلب HTTP][] (للملفات التي تجلبها من موقع ويب)، أو [عقدة ويب هوك][] (للملفات التي يتم إرسالها إلى سير عملك من مكان آخر)، أو من مصدر محلي. غالبًا ما تكون البيانات التي يتم الحصول عليها بهذه الطريقة بتنسيق ثنائي، على سبيل المثال، جدول بيانات أو ملف PDF.

تقوم عقدة Extract From File (استخراج من ملف) باستخراج البيانات من ملف بتنسيق ثنائي وتحويلها إلى JSON، والذي يمكن بعد ذلك معالجته بسهولة بواسطة بقية سير عملك. لتحويل JSON مرة أخرى إلى نوع ملف ثنائي، يرجى مراجعة [عقدة Convert to File (التحويل إلى ملف)](/integrations/builtin/core-nodes/n8n-nodes-base.converttofile.md).

## العمليات

استخدم القائمة المنسدلة **العمليات** لتحديد تنسيق الملف المصدر لاستخراج البيانات منه.

-   **Extract From CSV**: يُستخدم نوع الملف "القيم المفصولة بفواصل" (Comma Separated Values) بشكل شائع للبيانات المجدولة.
-   **Extract From HTML**: استخراج الحقول من ملفات بتنسيق HTML لصفحات الويب القياسية.
-   **Extract From JSON**: استخراج بيانات JSON من ملف ثنائي.
-   **Extract From ICS**: استخراج الحقول من ملفات بتنسيق iCalendar.
-   **Extract From ODS**: استخراج الحقول من ملفات جداول بيانات ODS.
-   **Extract From PDF**: استخراج الحقول من ملفات بتنسيق المستندات المحمولة (Portable Document Format).
-   **Extract From RTF**: استخراج الحقول من ملفات بتنسيق النص المنسق (Rich Text Format).
-   **Extract From Text File**: استخراج الحقول من تنسيق ملف نصي قياسي.
-   **Extract From XLS**: استخراج الحقول من ملف Microsoft Excel (التنسيق الأقدم).
-   **Extract From XLSX**: استخراج الحقول من ملف Microsoft Excel.
-   **Move File to Base64 String**: يحول البيانات الثنائية إلى تنسيق [base64][] سهل الاستخدام للنصوص.

## مثال على سير العمل

في هذا المثال، تُستخدم عقدة ويب هوك لتحفيز سير العمل. عند إرسال ملف CSV إلى عنوان الويب هوك، يتم إخراج بيانات الملف واستقبالها بواسطة عقدة Extract From File.

عند ضبطها للعمل كـ 'Extract from CSV'، تقوم العقدة بعد ذلك بإخراج البيانات كسلسلة من كائنات JSON من نوع 'row':

```
{
  "row": {
  "0": "apple",
  "1": "1",
  "2": "2",
  "3": "3"
  }
  ...
```

> **استقبال الملفات باستخدام ويب هوك**
>
> حدد زر **إضافة خيارات** الخاص بعقدة ويب هوك واختر **Raw body**، ثم قم بتمكين هذا الإعداد لجعل العقدة تُخرج الملف الثنائي الذي تتوقعه العقدة اللاحقة.

## معاملات العقدة

### حقل الإدخال الثنائي

أدخل اسم الحقل من بيانات إدخال العقدة الذي يحتوي على الملف الثنائي. القيمة الافتراضية هي 'data'.

### حقل الإخراج الوجهة

أدخل اسم الحقل في مخرجات العقدة الذي سيحتوي على البيانات المستخرجة.

يتوفر هذا المعامل فقط لهذه العمليات:

-   Extract From JSON
-   Extract From ICS
-   Extract From Text File
-   Move File to Base64 String
