## الاستخدام

كيفية استخدام عقدة Code.

### اختيار وضع

يوجد وضعان:

*   **التشغيل مرة واحدة لجميع العناصر (Run Once for All Items)**: هذا هو الوضع الافتراضي. عندما يتم تشغيل سير العمل الخاص بك، يتم تنفيذ الكود في عقدة Code مرة واحدة، بغض النظر عن عدد عناصر المدخلات الموجودة.
*   **التشغيل مرة واحدة لكل عنصر (Run Once for Each Item)**: اختر هذا إذا كنت تريد تشغيل الكود الخاص بك لكل عنصر مدخل.

## JavaScript

تدعم عقدة Code بيئة Node.js.

### ميزات JavaScript المدعومة

تدعم عقدة Code ما يلي:

*   الوعود (Promises). بدلاً من إرجاع العناصر مباشرةً، يمكنك إرجاع وعد (promise) يتم حله وفقًا لذلك.
*   الكتابة إلى وحدة تحكم المتصفح الخاص بك باستخدام `console.log`. هذا مفيد لتصحيح الأخطاء واستكشاف المشكلات في سير العمل الخاص بك.

### المكتبات الخارجية

إذا كنت تستضيف n8n ذاتيًا، يمكنك استيراد واستخدام وحدات npm المضمنة والخارجية في عقدة Code. لمعرفة كيفية تمكين الوحدات الخارجية، ارجع إلى دليل [تمكين الوحدات في عقدة Code](/hosting/configuration/configuration-examples/modules-in-code-node.md).

إذا كنت تستخدم n8n Cloud، فلا يمكنك استيراد وحدات npm خارجية. توفر n8n وحدتين لك:

*   [وحدة crypto لـ Node.js](https://nodejs.org/docs/latest-v18.x/api/crypto.html)
*   [حزمة moment npm](https://www.npmjs.com/package/moment)

### الأساليب والمتغيرات المضمنة

توفر n8n أساليب ومتغيرات مضمنة للتعامل مع البيانات والوصول إلى بيانات n8n. للمزيد من المعلومات، ارجع إلى [الأساليب والمتغيرات المضمنة](/code/builtin/overview.md).

صيغة استخدام الأساليب والمتغيرات المضمنة هي `$variableName` أو `$methodName()`. اكتب `
# Code node

` في عقدة الكود (Code node) أو محرر التعبيرات (expressions editor) لرؤية قائمة بالأساليب والمتغيرات المقترحة.

### اختصارات لوحة المفاتيح

تدعم بيئة تحرير عقدة الكود (Code node) اختصارات لوحة مفاتيح مفيدة وموفرة للوقت لمجموعة من العمليات، بدءًا من الإكمال التلقائي وطي الكود وصولاً إلى استخدام مؤشرات متعددة. اطلع على القائمة الكاملة لـ [اختصارات لوحة المفاتيح](/integrations/builtin/core-nodes/n8n-nodes-base.code/keyboard-shortcuts.md).

## بايثون (Pyodide - قديم)

Pyodide هي ميزة قديمة. لم تعد n8n v2 تدعم هذه الميزة.

أضافت n8n دعم بايثون في الإصدار 1.0. لا تتضمن ملفًا تنفيذيًا لبايثون. بدلاً من ذلك، توفر n8n دعم بايثون باستخدام [Pyodide](https://pyodide.org/en/stable/)، وهو منفذ لـ CPython إلى WebAssembly. يحد هذا من حزم بايثون المتاحة لتلك [المتضمنة مع Pyodide](https://pyodide.org/en/stable/usage/packages-in-pyodide.html#packages-in-pyodide). تقوم n8n بتنزيل الحزمة تلقائيًا في المرة الأولى التي تستخدمها فيها.

> **أبطأ من JavaScript**
>
> تستغرق عقدة الكود (Code node) وقتًا أطول لمعالجة بايثون مقارنةً بـ JavaScript. يرجع ذلك إلى خطوات الترجمة الإضافية.

### الأساليب والمتغيرات المضمنة

توفر n8n أساليب ومتغيرات مضمنة للتعامل مع البيانات والوصول إلى بيانات n8n. للمزيد من المعلومات، ارجع إلى [الأساليب والمتغيرات المضمنة](/code/builtin/overview.md).

صيغة استخدام الأساليب والمتغيرات المضمنة هي `_variableName` أو `_methodName()`. اكتب `_` في عقدة الكود (Code node) لرؤية قائمة بالأساليب والمتغيرات المقترحة.

### اختصارات لوحة المفاتيح

تدعم بيئة تحرير عقدة الكود (Code node) اختصارات لوحة مفاتيح مفيدة وموفرة للوقت لمجموعة من العمليات، بدءًا من الإكمال التلقائي وطي الكود وصولاً إلى استخدام مؤشرات متعددة. اطلع على القائمة الكاملة لـ [اختصارات لوحة المفاتيح](/integrations/builtin/core-nodes/n8n-nodes-base.code/keyboard-shortcuts.md).

## بايثون (أصلي)

أضاف n8n دعم بايثون الأصلي باستخدام مُشغّلات المهام في الإصدار 1.111.0. هذه الميزة مستقرة اعتبارًا من n8n v2.

الاختلافات الرئيسية عن Pyodide:

-   يدعم بايثون الأصلي `_items` فقط في وضع جميع العناصر و `_item` في وضع كل عنصر. ولا يدعم أساليب ومتغيرات n8n المدمجة الأخرى.
-   في الاستضافة الذاتية، يدعم بايثون الأصلي استيراد وحدات بايثون الأصلية من المكتبة القياسية ومن الأطراف الثالثة، إذا كانت صورة `n8nio/runners` تتضمنها وتسمح بها صراحةً. راجع [إضافة تبعيات إضافية لمُشغّلات المهام](/hosting/configuration/task-runners.md/#adding-extra-dependencies) لمزيد من التفاصيل.
-   يرفض بايثون الأصلي المدمجات غير الآمنة افتراضيًا. راجع [متغيرات بيئة مُشغّلات المهام](/hosting/configuration/environment-variables/task-runners.md) لمزيد من التفاصيل.
-   على عكس Pyodide، الذي يقبل تدوين الوصول بالنقطة، على سبيل المثال، `item.json.myNewField`، يقبل بايثون الأصلي فقط تدوين الوصول بالأقواس، على سبيل المثال، `item["json"]["my_new_field"]`. قد تكون هناك اختلافات طفيفة أخرى في بناء الجملة حيث يقبل Pyodide تراكيب غير قانونية في بايثون الأصلي.
-   على سحابة n8n، لا يسمح خيار بايثون لعقدة Code للمستخدمين باستيراد أي مكتبات بايثون — سواء من المكتبة القياسية أو حزم الأطراف الثالثة. يمكن لمستخدمي الاستضافة الذاتية العثور على تعليمات الإعداد لتضمين المكتبات الخارجية [هنا](https://docs.n8n.io/hosting/configuration/task-runners/#adding-extra-dependencies). على المدى الطويل، يلتزم فريق n8n بالسماح للمستخدمين بتنفيذ كود بايثون عشوائي بأمان مع أي مكتبات من الطرف الأول والثالث باستخدام مُشغّلات المهام.

الترقية إلى بايثون الأصلي هي تغيير جذري، لذلك قد تحتاج إلى تعديل نصوص بايثون البرمجية الخاصة بك لاستخدام مُشغّل بايثون الأصلي.

## البرمجة في n8n

هناك مكانان يمكنك فيهما استخدام الكود في n8n: عقدة Code ومحرر التعبيرات. عند استخدام أي من المنطقتين، هناك بعض المفاهيم الأساسية التي تحتاج إلى معرفتها، بالإضافة إلى بعض الأساليب والمتغيرات المدمجة للمساعدة في المهام الشائعة.

### المفاهيم الأساسية

عند العمل مع عقدة Code (التعليمات البرمجية)، تحتاج إلى فهم المفاهيم التالية:

*   [بنية البيانات](/data/data-structure.md): فهم البيانات التي تتلقاها في عقدة Code، والمتطلبات الخاصة بإخراج البيانات من العقدة.
*   [ربط العناصر](/data/data-mapping/data-item-linking/index.md): تعلم كيفية عمل عناصر البيانات، وكيفية ربطها بالعناصر من العقد السابقة. تحتاج إلى التعامل مع ربط العناصر في التعليمات البرمجية الخاصة بك عندما لا يتطابق عدد عناصر المدخلات والمخرجات.

### الدوال والمتغيرات المضمنة

تتضمن n8n دوالاً ومتغيرات مضمنة. توفر هذه الدعم لما يلي:

*   الوصول إلى بيانات عنصر محدد
*   الوصول إلى بيانات حول سير العمل، وعمليات التنفيذ، وبيئة n8n الخاصة بك
*   متغيرات مساعدة للتعامل مع البيانات والوقت

ارجع إلى [الدوال والمتغيرات المضمنة](/code/builtin/overview.md) لمزيد من المعلومات.

## استخدام الذكاء الاصطناعي في عقدة Code

> **توفر الميزة**
>
> تتوفر مساعدة الذكاء الاصطناعي في عقدة Code لمستخدمي السحابة. وهي غير متوفرة في n8n المستضافة ذاتيًا.

> **التعليمات البرمجية التي يولدها الذكاء الاصطناعي ستحل محل التعليمات البرمجية الخاصة بك**
>
> إذا كنت قد كتبت بالفعل بعض التعليمات البرمجية في علامة التبويب **Code**، فستحل التعليمات البرمجية التي يولدها الذكاء الاصطناعي محلها. توصي n8n باستخدام الذكاء الاصطناعي كنقطة بداية لإنشاء التعليمات البرمجية الأولية الخاصة بك، ثم تعديلها حسب الحاجة.

لاستخدام ChatGPT لتوليد التعليمات البرمجية في عقدة Code:

1.  في عقدة Code، اضبط **اللغة (Language)** على **JavaScript**.
2.  حدد علامة التبويب **اسأل الذكاء الاصطناعي (Ask AI)**.
3.  اكتب الاستعلام الخاص بك.
4.  حدد **توليد التعليمات البرمجية (Generate Code)**. ترسل n8n الاستعلام الخاص بك إلى ChatGPT، ثم تعرض النتيجة في علامة التبويب **Code**.

# المشكلات الشائعة في عقدة Code

فيما يلي بعض الأخطاء والمشكلات الشائعة المتعلقة بـ [عقدة Code](/integrations/builtin/core-nodes/n8n-nodes-base.code/index.md) وخطوات حلها أو استكشافها.

## التعليمات البرمجية لا تُرجع العناصر بشكل صحيح

يحدث هذا الخطأ عندما لا تُرجع التعليمات البرمجية في عقدة Code البيانات بالتنسيق المتوقع.

في n8n، جميع البيانات التي يتم تمريرها بين العقد هي مصفوفة من الكائنات. يغلف كل كائن من هذه الكائنات كائنًا آخر بمفتاح `json`:

```javascript
[
  {
    "json": {
	  // your data goes here
	}
  }
]
```

لاستكشاف هذا الخطأ وإصلاحه، تحقق مما يلي:

*   اقرأ [بنية البيانات](/data/data-structure.md) لفهم البيانات التي تتلقاها في عقدة Code والمتطلبات الخاصة بإخراج البيانات من العقدة.
*   افهم كيفية عمل عناصر البيانات وكيفية ربط عناصر البيانات من العقد السابقة باستخدام [ربط العناصر](/data/data-mapping/data-item-linking/index.md).

## خاصية 'json' ليست كائنًا
<!-- vale on -->

يحدث هذا الخطأ عندما تُرجع العقدة (Code node) بيانات حيث لا يشير المفتاح `json` إلى كائن.

قد يحدث هذا إذا قمت بتعيين `json` لهيكل بيانات مختلف، مثل مصفوفة:

```javascript
[
  {
    "json": [
	  // تعيين `json` لمصفوفة كهذه سيؤدي إلى حدوث خطأ
	]
  }
]
```

لحل هذه المشكلة، تأكد من أن المفتاح `json` يشير إلى كائن في بيانات الإرجاع الخاصة بك:

```javascript
[
  {
    "json": {
	  // تعيين `json` لكائن كما هو متوقع
	}
  }
]
```

## الكود لا يُرجع كائنًا

قد يحدث هذا الخطأ عندما لا تُرجع العقدة (Code node) أي شيء أو إذا أرجعت نتيجة غير متوقعة.

لحل هذه المشكلة، تأكد من أن العقدة (Code node) الخاصة بك تُرجع [هيكل البيانات المتوقع](/data/data-structure.md):

```javascript
[
  {
    "json": {
	  // بياناتك هنا
	}
  }
]
```

قد يحدث هذا الخطأ أيضًا إذا كان الكود الذي قدمته يُرجع `'undefined'` بدلاً من النتيجة المتوقعة. في هذه الحالة، تأكد من أن البيانات التي تشير إليها في العقدة (Code node) موجودة في كل تنفيذ وأنها تحتوي على الهيكل الذي يتوقعه الكود الخاص بك.

## 'import' و 'export' يجب أن تظهر فقط في المستوى الأعلى

يحدث هذا الخطأ إذا حاولت استخدام `import` أو `export` في العقدة (Code node). هذه ليست مدعومة بواسطة بيئة JavaScript المعزولة (sandbox) في n8n. بدلاً من ذلك، استخدم الدالة `require` لتحميل الوحدات (modules).

لحل هذه المشكلة، حاول تغيير عبارات `import` الخاصة بك لاستخدام `require`:

```javascript
// الكود الأصلي:
// import express from "express";
// الكود الجديد:
const express = require("express");
```

## لا يمكن العثور على الوحدة '&lt;module&gt;'
<!-- vale on -->

يحدث هذا الخطأ إذا حاولت استخدام `require` في العقدة (Code node) ولم يتمكن n8n من العثور على الوحدة (module).

> **للاستضافة الذاتية فقط**
>
> لا يدعم n8n استيراد الوحدات (modules) في إصدار [السحابة](/manage-cloud/overview.md).

إذا كنت [تستضيف n8n ذاتيًا](/hosting/index.md)، فاتبع هذه الخطوات:

* قم بتثبيت الوحدة (module) في بيئة n8n الخاصة بك.
	* إذا كنت تقوم بتشغيل n8n باستخدام [npm](/hosting/installation/npm.md)، فقم بتثبيت الوحدة (module) في نفس بيئة n8n.
	* إذا كنت تقوم بتشغيل n8n باستخدام [Docker](/hosting/installation/docker.md)، فأنت بحاجة إلى توسيع صورة n8n الرسمية بصورة [مخصصة](https://docs.docker.com/build/building/base-images/) تتضمن الوحدة (module) الخاصة بك.
* قم بتعيين [متغيرات البيئة](/hosting/configuration/configuration-examples/modules-in-code-node.md) `NODE_FUNCTION_ALLOW_BUILTIN` و `NODE_FUNCTION_ALLOW_EXTERNAL` للسماح باستيراد الوحدات (modules).

## استخدام المتغيرات العامة

قد ترغب أحيانًا في تعيين واسترداد بيانات عامة بسيطة متعلقة بسير عمل عبر عمليات التنفيذ وداخلها. على سبيل المثال، قد ترغب في تضمين تاريخ التقرير السابق عند تجميع تقرير يتضمن قائمة بتحديثات المشروع.

لتعيين وتحديث واسترداد البيانات مباشرةً إلى سير عمل، استخدم دوال [البيانات الثابتة](/code/cookbook/builtin/get-workflow-static-data.md) ضمن الكود الخاص بك. يمكنك إدارة البيانات إما بشكل عام أو ربطها بعقد محددة.

> **استخدم عقدة إزالة التكرارات (Remove Duplicates) عندما يكون ذلك ممكنًا**
>
> إذا كنت مهتمًا باستخدام المتغيرات لتجنب معالجة نفس عناصر البيانات (items) أكثر من مرة، ففكر في استخدام [عقدة إزالة التكرارات (Remove Duplicates)](/integrations/builtin/core-nodes/n8n-nodes-base.removeduplicates/index.md) بدلاً من ذلك. يمكن لعقدة إزالة التكرارات (Remove Duplicates) حفظ المعلومات عبر عمليات التنفيذ لتجنب معالجة نفس العناصر (items) عدة مرات.

## لا يمكن الوصول إلى بيانات الاعتماد (credentials) في عقدة الكود (Code node)

حسب التصميم، لا يمكن لعقد الكود (Code nodes) الوصول إلى بيانات الاعتماد (credentials). ليس لديها وصول إلى نظام n8n الداخلي لإدارة بيانات الاعتماد (credential management system). يمنع هذا الكشف عن بيانات المصادقة (authentication) الحساسة.

محاولات الإشارة إلى بيانات الاعتماد (credentials) في عقدة الكود (Code node) باستخدام التعبيرات (expressions) أو دوال مثل `this.getCredentials()` أو `$getCredentials()` ستؤدي إلى أخطاء (errors)، مثل `this.getCredentials is not a function` و`$getCredentials is not defined`.

إذا كنت بحاجة إلى إجراء استدعاءات API مصادق عليها، فاستخدم [عقدة طلب HTTP (HTTP Request)](/integrations/builtin/core-nodes/n8n-nodes-base.httprequest/index.md) التي توفر دعمًا لبيانات الاعتماد (credentials).

للتعامل مع بيانات الاعتماد (credentials) ديناميكيًا، تعامل مع منطق اختيار بيانات الاعتماد (credentials) خارج عقدة الكود (Code node):

- استخدم [عقدة التبديل (Switch)](/integrations/builtin/core-nodes/n8n-nodes-base.switch.md) لتوجيه سير العمل إلى عقد مختلفة ببيانات اعتماد (credentials) مختلفة.
- استخدم التعبيرات (expressions) مباشرةً في حقول بيانات الاعتماد (credentials) لاختيار بيانات الاعتماد (credentials) ديناميكيًا بناءً على بيانات العقدة السابقة.
- استخدم عقدة طلب HTTP (HTTP Request) مع مصادقة مخصصة (Custom Auth) لتعيين الرؤوس (headers) أو معاملات الاستعلام (query parameters) أو قيم الجسم (body values) ديناميكيًا باستخدام التعبيرات (expressions).
