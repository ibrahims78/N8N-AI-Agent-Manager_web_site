# تنفيذ الأمر

تقوم العقدة (Execute Command) بتشغيل أوامر shell على الجهاز المضيف الذي يشغل n8n.

> **اعتبارات أمنية**
>
> يمكن لعقدة (Execute Command) أن تُدخل مخاطر أمنية كبيرة في البيئات التي تعمل مع مستخدمين غير موثوق بهم. لهذا السبب، يتم [تعطيل](/hosting/securing/blocking-nodes.md#exclude-nodes) العقدة افتراضيًا بدءًا من الإصدار 2.0.

> **أي shell يقوم بتشغيل الأمر؟**
>
> تقوم هذه العقدة بتنفيذ الأمر في الـ shell الافتراضي للجهاز المضيف. على سبيل المثال، `cmd` على Windows و `zsh` على macOS.
>
> إذا قمت بتشغيل n8n باستخدام Docker، فسيتم تشغيل أمرك في حاوية n8n وليس في مضيف Docker.
>
> إذا كنت تستخدم [وضع قائمة الانتظار](/hosting/scaling/queue-mode.md)، يتم تشغيل الأمر على العامل الذي يقوم بتنفيذ المهمة في وضع الإنتاج. عند تشغيل عمليات التنفيذ اليدوية، يتم تشغيله على النسخة الرئيسية، ما لم تقم بتعيين `OFFLOAD_MANUAL_EXECUTIONS_TO_WORKERS` إلى `true`.

> **غير متاح على السحابة**
>
> هذه العقدة غير متاحة على n8n Cloud.

## معاملات العقدة

قم بتكوين العقدة باستخدام المعاملات التالية.

### التنفيذ مرة واحدة

اختر ما إذا كنت تريد أن يتم تنفيذ العقدة مرة واحدة فقط (مفعل) أو مرة واحدة لكل عنصر تتلقاه كمدخلات (معطل).

### الأمر

أدخل الأمر المراد تنفيذه على الجهاز المضيف. ارجع إلى الأقسام أدناه للاطلاع على أمثلة لتشغيل [أوامر متعددة](#run-multiple-commands) و [أوامر cURL](#run-curl-command).

#### تشغيل أوامر متعددة

استخدم إحدى طريقتين لتشغيل أوامر متعددة في عقدة (Execute Command) واحدة:

*   أدخل كل أمر على سطر واحد مفصولاً بـ `&&`. على سبيل المثال، يمكنك دمج أمر تغيير الدليل (cd) مع أمر القائمة (ls) باستخدام `&&`.

    ```bash
    cd bin && ls
    ```

*   أدخل كل أمر على سطر منفصل. على سبيل المثال، يمكنك كتابة أمر القائمة (ls) على سطر جديد بعد أمر تغيير الدليل (cd).

    ```bash
    cd bin
    ls
    ```

#### تشغيل أمر cURL

يمكنك أيضًا استخدام عقدة [HTTP Request](/integrations/builtin/core-nodes/n8n-nodes-base.httprequest/index.md) لإجراء طلب cURL.

إذا كنت ترغب في تشغيل أمر curl في عقدة (Execute Command)، فسيتعين عليك بناء صورة Docker استنادًا إلى صورة n8n الموجودة. تستخدم صورة Docker الافتراضية لـ n8n نظام Alpine Linux. سيتعين عليك تثبيت حزمة curl.

1.  أنشئ ملفًا باسم `Dockerfile`.
2.  أضف مقتطف الكود أدناه إلى ملف Dockerfile.

    ```shell
    FROM docker.n8n.io/n8nio/n8n
    USER root
    RUN apk --update add curl
    USER node
    ```

3.  في نفس المجلد، نفّذ الأمر أدناه لبناء صورة Docker.

    ```shell
    docker build -t n8n-curl
    ```

4.  استبدل صورة Docker التي استخدمتها من قبل. على سبيل المثال، استبدل `docker.n8n.io/n8nio/n8n` بـ `n8n-curl`.
5.  قم بتشغيل صورة Docker التي تم إنشاؤها حديثًا. ستتمكن الآن من تنفيذ ssh باستخدام عقدة (Execute Command).

## Command failed: &lt;command&gt; /bin/sh: &lt;command&gt;: not found
<!-- vale on -->

يحدث هذا الخطأ عندما لا تستطيع بيئة الشل العثور على أحد الأوامر في المعامل **Command**.

لإصلاح هذا الخطأ، راجع ما يلي:

*   تحقق من أن الأمر ووسائطه لا تحتوي على أخطاء إملائية في المعامل **Command**.
*   تحقق من أن الأمر موجود في `PATH` الخاص بالمستخدم الذي يقوم بتشغيل n8n.
*   إذا كنت تقوم بتشغيل n8n باستخدام Docker، فتحقق مما إذا كان الأمر متاحًا داخل الحاوية بمحاولة تشغيله يدويًا. إذا لم يكن أمرك متضمنًا في الحاوية، فقد تحتاج إلى توسيع صورة n8n الرسمية باستخدام [صورة مخصصة](https://docs.docker.com/build/building/base-images/) تتضمن أمرك.
    *   إذا كان n8n قيد التشغيل بالفعل:
        ```sh
        # ابحث عن معرّف حاوية n8n، سيكون العمود الأول
        docker ps | grep n8n
        # حاول تنفيذ الأمر داخل الحاوية قيد التشغيل
        docker container exec <container_ID> <command_to_run>
        ```
    *   إذا لم يكن n8n قيد التشغيل:
        ```sh
        # ابدأ تشغيل حاوية جديدة تقوم بتشغيل الأمر بدلاً من n8n
        # استخدم نفس الصورة والعلامة التي تستخدمها لتشغيل n8n بشكل طبيعي
        docker run -it --rm --entrypoint /bin/sh docker.n8n.io/n8nio/n8n -c <command_to_run>
        ```

## Error: stdout maxBuffer length exceeded
<!-- vale on -->

يحدث هذا الخطأ عندما يعيد أمرك مخرجات أكثر مما تستطيع عقدة Execute Command معالجته في وقت واحد.

لتجنب هذا الخطأ، قلل المخرجات التي ينتجها أمرك. راجع صفحة الدليل أو وثائق أمرك لمعرفة ما إذا كانت هناك علامات لتقييد أو تصفية المخرجات. إذا لم يكن الأمر كذلك، فقد تحتاج إلى توجيه المخرجات إلى أمر آخر لإزالة المعلومات غير الضرورية.
