# عقدة Discord

استخدم عقدة Discord لأتمتة سير العمل في Discord، ودمج Discord مع تطبيقات أخرى. تدعم n8n مجموعة واسعة من ميزات Discord، بما في ذلك إرسال الرسائل في قناة Discord وإدارة القنوات.

ستجد في هذه الصفحة قائمة بالعمليات التي تدعمها عقدة Discord وروابط لموارد إضافية.

> **بيانات الاعتماد**
>
> يُرجى الرجوع إلى [بيانات اعتماد Discord](/integrations/builtin/credentials/discord.md) للحصول على إرشادات حول إعداد المصادقة.

> **يمكن استخدام هذه العقدة كأداة للذكاء الاصطناعي**
>
> يمكن استخدام هذه العقدة لتعزيز قدرات وكيل الذكاء الاصطناعي. عند استخدامها بهذه الطريقة، يمكن تعيين العديد من المعاملات تلقائيًا، أو باستخدام معلومات موجهة بواسطة الذكاء الاصطناعي - اكتشف المزيد في [وثائق معاملات أداة الذكاء الاصطناعي](/advanced-ai/examples/using-the-fromai-function.md).

> **التدخل البشري لاستدعاءات أدوات الذكاء الاصطناعي**
>
> يمكن استخدام هذه العقدة كخطوة مراجعة بشرية لاستدعاءات أدوات وكيل الذكاء الاصطناعي. عند تكوينها بهذه الطريقة، سيتوقف وكيل الذكاء الاصطناعي ويطلب موافقة بشرية عبر هذه الخدمة قبل تنفيذ الأدوات التي تتطلب إشرافًا. تعرف على المزيد في [التدخل البشري لاستدعاءات أدوات الذكاء الاصطناعي](/advanced-ai/human-in-the-loop-tools.md).

## الانتظار لاستجابة

باختيار عملية **إرسال وانتظار استجابة**، يمكنك إرسال رسالة وإيقاف تنفيذ سير العمل مؤقتًا حتى يؤكد شخص ما الإجراء أو يقدم المزيد من المعلومات.

### نوع الاستجابة

يمكنك الاختيار بين الأنواع التالية من إجراءات الانتظار والموافقة:

*   **الموافقة**: يمكن للمستخدمين الموافقة أو عدم الموافقة من داخل الرسالة.
*   **نص حر**: يمكن للمستخدمين إرسال استجابة باستخدام نموذج.
*   **نموذج مخصص**: يمكن للمستخدمين إرسال استجابة باستخدام نموذج مخصص.

يمكنك تخصيص سلوك الانتظار والاستجابة اعتمادًا على نوع الاستجابة الذي تختاره. يمكنك تكوين هذه الخيارات في أي من أنواع الاستجابة المذكورة أعلاه:

*   **تحديد مهلة الانتظار**: ما إذا كان سير العمل سيستأنف التنفيذ تلقائيًا بعد مهلة زمنية محددة. يمكن أن تكون هذه المهلة فترة زمنية أو وقتًا محددًا.
*   **إلحاق إسناد n8n**: ما إذا كان سيتم ذكر في الرسالة أنها أُرسلت تلقائيًا باستخدام n8n (مفعل) أم لا (معطل).

### تخصيص استجابة الموافقة

عند استخدام نوع استجابة الموافقة، يمكنك اختيار ما إذا كنت تريد تقديم زر موافقة فقط أو زري موافقة *ورفض* معًا.

يمكنك أيضًا تخصيص تسميات الأزرار للأزرار التي تقوم بتضمينها.

### تخصيص استجابة النص الحر

عند استخدام نوع استجابة النص الحر، يمكنك تخصيص تسمية زر الرسالة، وعنوان النموذج ووصفه، وتسمية زر الاستجابة.

### تخصيص استجابة النموذج المخصص

عند استخدام نوع استجابة النموذج المخصص، يمكنك إنشاء نموذج باستخدام الحقول والخيارات التي تريدها.

يمكنك تخصيص كل عنصر من عناصر النموذج باستخدام الإعدادات الموضحة في [عناصر نموذج مُحفِّز n8n Form](/integrations/builtin/core-nodes/n8n-nodes-base.formtrigger.md#form-elements). لإضافة المزيد من الحقول، حدد زر **إضافة عنصر نموذج**.

ستتمكن أيضًا من تخصيص تسمية زر الرسالة، وعنوان النموذج ووصفه، وتسمية زر الاستجابة.

## ماذا تفعل إذا كانت عمليتك غير مدعومة

إذا كانت هذه العقدة لا تدعم العملية التي تريد القيام بها، يمكنك استخدام [عقدة HTTP Request](/integrations/builtin/core-nodes/n8n-nodes-base.httprequest/index.md) لاستدعاء API الخاص بالخدمة.

يمكنك استخدام بيانات الاعتماد التي أنشأتها لهذه الخدمة في عقدة HTTP Request:

1.  في عقدة HTTP Request، حدد **المصادقة** > **نوع بيانات الاعتماد المُحددة مسبقًا**.
2.  حدد الخدمة التي تريد الاتصال بها.
3.  حدد بيانات الاعتماد الخاصة بك.

ارجع إلى [عمليات API المخصصة](/integrations/custom-operations.md) لمزيد من المعلومات.

# المشكلات الشائعة لعقدة Discord

فيما يلي بعض الأخطاء والمشكلات الشائعة المتعلقة بـ [عقدة Discord (ديسكورد)](/integrations/builtin/app-nodes/n8n-nodes-base.discord/index.md) وخطوات حلها أو استكشافها.

## إضافة حقول إضافية إلى التضمينات

يمكن أن تتضمن رسائل Discord اختياريًا تضمينات (embeds)، وهي مكون معاينة غني يمكن أن يشمل عنوانًا ووصفًا وصورة ورابطًا والمزيد.

تدعم عقدة Discord التضمينات عند استخدام العملية **Send** على المورد **Message**. حدد **Add Embeds** لتعيين حقول إضافية تتضمن Description و Author و Title و URL و URL Image.

لإضافة حقول غير متضمنة افتراضيًا، اضبط **Input Method** على **Raw JSON**. من هنا، أضف كائن JSON إلى المعامل **Value** الذي يحدد [أسماء الحقول](https://discord.com/developers/docs/resources/message#embed-object) والقيم التي ترغب في تضمينها.

على سبيل المثال، لتضمين `footer` و `fields`، وكلاهما غير متاح باستخدام **Enter Fields** Input Method، يمكنك استخدام كائن JSON بهذا الشكل:

```json
{
    "author": "My Name",
	"url": "https://discord.js.org",
	"fields": [
		{
			"name": "Regular field title",
			"value": "Some value here"
		}
	],
	"footer": {
		"text": "Some footer text here",
		"icon_url": "https://i.imgur.com/AfFp7pu.png"
	}
}
```

يمكنك معرفة المزيد حول التضمينات في [استخدام الويب هوكس والتضمينات | Discord](https://discord.com/safety/using-webhooks-and-embeds).

إذا واجهت مشكلات عند العمل مع التضمينات باستخدام عقدة Discord، يمكنك استخدام [HTTP Request](/integrations/builtin/core-nodes/n8n-nodes-base.httprequest/index.md) مع بيانات اعتماد Discord الحالية لديك لإجراء `POST` إلى عنوان URL التالي:

```
https://discord.com/api/v10/channels/<CHANNEL_ID>/messages
```

في نص الطلب، قم بتضمين معلومات التضمين الخاصة بك في محتوى الرسالة بهذا الشكل:

```json
{
	"content": "Test",
	"embeds": [
		{
			"author": "My Name",
			"url": "https://discord.js.org",
			"fields": [
				{
					"name": "Regular field title",
					"value": "Some value here"
				}
			],
			"footer": {
				"text": "Some footer text here",
				"icon_url": "https://i.imgur.com/AfFp7pu.png"
			}
		}
	]
}
```

## الإشارة إلى المستخدمين والقنوات

للإشارة إلى المستخدمين والقنوات في رسائل Discord، تحتاج إلى تنسيق رسالتك وفقًا [لإرشادات تنسيق الرسائل الخاصة بـ Discord](https://discord.com/developers/docs/reference#message-formatting).

للإشارة إلى مستخدم، تحتاج إلى معرفة معرف المستخدم (user ID) الخاص بمستخدم Discord. ضع في اعتبارك أن معرف المستخدم يختلف عن اسم العرض الخاص بالمستخدم. وبالمثل، تحتاج إلى معرف قناة (channel ID) للربط بقناة معينة.

يمكنك معرفة كيفية تمكين وضع المطور ونسخ معرفات المستخدم أو القناة في [توثيق Discord حول العثور على معرفات المستخدم/الخادم/الرسالة](https://support.discord.com/hc/en-us/articles/206346498-Where-can-I-find-my-User-Server-Message-ID).

بمجرد حصولك على معرف المستخدم أو القناة، يمكنك تنسيق رسالتك باستخدام الصيغة التالية:

*   **المستخدم**: `<@USER_ID>`
*   **القناة**: `<#CHANNEL_ID>`
*   **الدور**: `<@&ROLE_ID>`
