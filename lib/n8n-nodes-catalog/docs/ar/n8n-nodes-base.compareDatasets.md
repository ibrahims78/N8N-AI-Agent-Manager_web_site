# Compare Datasets (مُقارن البيانات)

تساعد عقدة Compare Datasets في مقارنة البيانات الناتجة من تياريْن إدخال.

## معاملات العقدة

1. قرر الحقول التي ستتم مقارنتها. في **Input A Field**، أدخل اسم الحقل الذي تريد استخدامه من تيار الإدخال A. في **Input B Field**، أدخل اسم الحقل الذي تريد استخدامه من تيار الإدخال B. 
2. **Optional**: يمكنك المقارنة عبر حقول متعددة. اختر **Add Fields to Match** لإعداد المزيد من المطابقات.
3. اختر كيفية التعامل مع الاختلافات بين مجموعتي البيانات. في **When There Are Differences**، اختر أحد الخيارات التالية:
	* **Use Input A Version** لتعامل تدفق الإدخال A كمصدر للحقيقة.
	* **Use Input B Version** لتعامل تدفق الإدخال B كمصدر للحقيقة.
	* **Use a Mix of Versions** لاستخدام مدخلات مختلفة لحقول مختلفة.
		* استخدم **Prefer** لتحديد إما **Input A Version** أو **Input B Version** كمصدر الحقيقة الرئيسي.
		* أدخل حقول الإدخال التي تُعتبر استثناءً لـ **For Everything Except** لسحبها من مصدر الإدخال الآخر. لإضافة عدة حقول إدخال، أدخل قائمة مفصولة بفواصل.
	* **Include Both Versions** لإدراج كلا تياري الإدخال في المخرجات، ما قد يجعل البنية أكثر تعقيداً.
4. قرر ما إذا كنت ستستخدم **Fuzzy Compare**. عند التشغيل، ستتسامح المقارنة مع فروقات بسيطة في النوع عند مقارنة الحقول. على سبيل المثال، يُعامل الرقم `3` كسلسلة `3` كقيم متطابقة عند تشغيل **Fuzzy Compare**، لكنه لن يُعامل كذلك عند إيقاف تشغيله.

## فهم مقارنة العناصر

تتم مقارنة العناصر في عمليتين مرحليتين:

1. تتحقق n8n مما إذا كانت قيم الحقول التي حددتها للمقارنة مطابقة عبر كلا الإدخالين.
2. إذا تطابقت الحقول للمقارنة، تقارن n8n بعد ذلك جميع الحقول داخل العناصر لتحديد ما إذا كانت العناصر متطابقة أم مختلفة.

## خيارات العقدة

استخدم العقدة **Options** لتحديد تفاصيل المقارنة أو تعديل سلوك المقارنة.

### الحقول التي سيتم تجاهلها في المقارنة

أدخل أسماء الحقول التي تريد تجاهلها أثناء المقارنة.

على سبيل المثال، إذا قمت بمقارنة مجموعتي البيانات أدناه باستخدام `person.language` كـ **Fields to Match**، ستُعاد البيانات على أنها مختلفة. إذا أضفت `person.name` إلى **Fields to Skip Comparing**، فستُعاد البيانات على أنها متطابقة.

```json
	// Input 1
	[
		{
			"person":
			{
				"name":	"Stefan",
				"language":	"de"
			}
		},
		{
			"person":
			{
				"name":	"Jim",
				"language":	"en"
			}
		},
		{
			"person":
			{
				"name":	"Hans",
				"language":	"de"
			}
		}
	]
	// Input 2
		[
		{
			"person":
			{
				"name":	"Sara",
				"language":	"de"
			}
		},
		{
			"person":
			{
				"name":	"Jane",
				"language":	"en"
			}
		},
		{
			"person":
			{
				"name":	"Harriet",
				"language":	"de"
			}
		}
	]
```

### تعطيل Dot Notation

ما إذا كان سيتم منع الإشارة إلى الحقول الفرعية باستخدام `parent.child` في اسم الحقل (مفعل) أم السماح بذلك (مُعطل، الافتراضي).

### التطابقات المتعددة

اختر كيفية التعامل مع البيانات المكررة. الافتراضي هو **Include All Matches**. يمكنك اختيار **Include First Match Only**.

على سبيل المثال، بالنظر إلى هذين المجموعتين من البيانات:
```json
	// Input 1
	[
		{
			"fruit": {
				"type": "apple",
				"color": "red"
			}
		},
				{
			"fruit": {
				"type": "apple",
				"color": "red"
			}
		},
				{
			"fruit": {
				"type": "banana",
				"color": "yellow"
			}
		}
	]
	// Input 2
	[
		{
			"fruit": {
				"type": "apple",
				"color": "red"
			}
		},
				{
			"fruit": {
				"type": "apple",
				"color": "red"
			}
		},
				{
			"fruit": {
				"type": "banana",
				"color": "yellow"
			}
		}
	]
```

تُعيد n8n ثلاثة عناصر في تبويب **Same Branch**. البيانات متطابقة في كلا الفرعين.

إذا قمت باختيار **Include First Match Only**، ستُعيد n8n عنصرين في تبويب **Same Branch**. البيانات متطابقة في كلا الفرعين، لكن n8n تعيد فقط أول ظهور لعناصر "apple" المطابقة.

## فهم المخرجات

هناك أربع خيارات للمخرجات:

* **In A only Branch**: يحتوي البيانات التي تحدث فقط في الإدخال الأول.
* **Same Branch**: يحتوي البيانات المطابقة في كلا الإدخالين.
* **Different Branch**: يحتوي البيانات المختلفة بين الإدخالين.
* **In B only Branch**: يحتوي البيانات التي تحدث فقط في الإدخال الثاني.
