# مقارنة مجموعات البيانات

تساعد عقدة مقارنة مجموعات البيانات (Compare Datasets) في مقارنة البيانات من تدفقين من المدخلات.

## معاملات العقدة

1.  حدد الحقول المراد مقارنتها. في **Input A Field**، أدخل اسم الحقل الذي تريد استخدامه من تدفق المدخلات A. في **Input B Field**، أدخل اسم الحقل الذي تريد استخدامه من تدفق المدخلات B.
2.  **اختياري**: يمكنك المقارنة بواسطة حقول متعددة. حدد **Add Fields to Match** لإعداد المزيد من المقارنات.
3.  اختر كيفية التعامل مع الاختلافات بين مجموعات البيانات. في **When There Are Differences**، حدد أحد الخيارات التالية:
    *   **Use Input A Version** للتعامل مع تدفق المدخلات A كمصدر للحقيقة.
    *   **Use Input B Version** للتعامل مع تدفق المدخلات B كمصدر للحقيقة.
    *   **Use a Mix of Versions** لاستخدام مدخلات مختلفة لحقول مختلفة.
        *   استخدم **Prefer** لتحديد إما **Input A Version** أو **Input B Version** كمصدر رئيسي للحقيقة.
        *   أدخل حقول المدخلات التي تمثل استثناءات لـ **For Everything Except** للسحب من مصدر المدخلات الآخر. لإضافة حقول مدخلات متعددة، أدخل قائمة مفصولة بفواصل.
    *   **Include Both Versions** لتضمين كلا تدفقي المدخلات في المخرجات، مما قد يجعل البنية أكثر تعقيدًا.
4.  حدد ما إذا كنت تريد استخدام **Fuzzy Compare**. عند تفعيله، ستتسامح المقارنة مع اختلافات الأنواع الصغيرة عند مقارنة الحقول. على سبيل المثال، يتم التعامل مع الرقم 3 والسلسلة النصية `3` على أنهما متطابقان عند تفعيل **Fuzzy Compare**، ولكن لن يتم التعامل معهما بنفس الطريقة عند إيقاف تشغيله.

## فهم مقارنة العناصر

مقارنة العناصر هي عملية من مرحلتين:

1.  تتحقق n8n مما إذا كانت قيم الحقول التي حددتها للمقارنة تتطابق عبر كلا المدخلين.
2.  إذا تطابقت الحقول المراد مقارنتها، تقارن n8n بعد ذلك جميع الحقول داخل العناصر، لتحديد ما إذا كانت العناصر متطابقة أم مختلفة.

## خيارات العقدة

استخدم خيارات العقدة (Node **Options**) لتحسين مقارنتك أو تعديل سلوك المقارنة.

### الحقول المراد تخطي مقارنتها

أدخل أسماء الحقول التي تريد تجاهلها في المقارنة.

على سبيل المثال، إذا قارنت مجموعتي البيانات أدناه باستخدام `person.language` كـ **Fields to Match**، ستعيد n8n إرجاعهما على أنهما مختلفان. إذا أضفت `person.name` إلى **Fields to Skip Comparing**، ستعيد n8n إرجاعهما على أنهما متطابقان.

```json
	// Input 1
	[
		{
			"person":
			{
				"name":	"Stefan",
				"language":	"de"
			}
		},
		{
			"person":
			{
				"name":	"Jim",
				"language":	"en"
			}
		},
		{
			"person":
			{
				"name":	"Hans",
				"language":	"de"
			}
		}
	]
	// Input 2
		[
		{
			"person":
			{
				"name":	"Sara",
			"language":	"de"
			}
		},
		{
			"person":
			{
				"name":	"Jane",
				"language":	"en"
			}
		},
		{
			"person":
			{
				"name":	"Harriet",
				"language":	"de"
			}
		}
	]
```

### تعطيل تدوين النقطة (Dot Notation)

يحدد ما إذا كان سيتم منع الإشارة إلى الحقول الفرعية باستخدام `parent.child` في اسم الحقل (مُفعّل) أو السماح بذلك (مُعطّل، الافتراضي).

### تطابقات متعددة

اختر كيفية التعامل مع البيانات المكررة. الافتراضي هو **تضمين جميع التطابقات**. يمكنك اختيار **تضمين أول تطابق فقط**.

على سبيل المثال، بالنظر إلى مجموعتي البيانات هاتين:
```json
	// Input 1
	[
		{
			"fruit": {
				"type": "apple",
				"color": "red"
			}
		},
				{
			"fruit": {
				"type": "apple",
				"color": "red"
			}
		},
				{
			"fruit": {
				"type": "banana",
				"color": "yellow"
			}
		}
	]
	// Input 2
	[
		{
			"fruit": {
				"type": "apple",
				"color": "red"
			}
		},
				{
			"fruit": {
				"type": "apple",
				"color": "red"
			}
		},
				{
			"fruit": {
				"type": "banana",
				"color": "yellow"
			}
		}
	]
```

تُرجع n8n ثلاثة عناصر في علامة التبويب **الفرع المتطابق**. البيانات متطابقة في كلا الفرعين.

إذا اخترت **تضمين أول تطابق فقط**، تُرجع n8n عنصرين في علامة التبويب **الفرع المتطابق**. البيانات متطابقة في كلا الفرعين، لكن n8n تُرجع فقط أول ظهور للعناصر المتطابقة "apple".

## فهم المخرجات

توجد أربعة خيارات للمخرجات:

*   **في فرع A فقط**: يحتوي على بيانات تظهر فقط في المدخلات الأولى.
*   **الفرع المتطابق**: يحتوي على بيانات متطابقة في كلا المدخلين.
*   **الفرع المختلف**: يحتوي على بيانات مختلفة بين المدخلات.
*   **في فرع B فقط**: يحتوي على بيانات تظهر فقط في المخرجات الثانية.
