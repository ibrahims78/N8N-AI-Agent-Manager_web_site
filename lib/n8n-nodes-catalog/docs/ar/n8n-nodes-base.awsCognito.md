# عقدة AWS Cognito

استخدم عقدة AWS Cognito (خدمة إدارة الهوية والوصول من أمازون) لأتمتة سير العمل في AWS Cognito وتكامل AWS Cognito مع التطبيقات الأخرى. تدعم n8n بشكل مدمج مجموعة واسعة من ميزات AWS Cognito، والتي تشمل إنشاء المجموعات والمستخدمين ومجمعات المستخدمين واستردادها وتحديثها وحذفها.

ستجد في هذه الصفحة قائمة بالعمليات التي تدعمها عقدة AWS Cognito، وروابط لموارد إضافية.

> **بيانات الاعتماد**
>
> يمكنك العثور على معلومات المصادقة لهذه العقدة [هنا](/integrations/builtin/credentials/aws.md).

## العمليات

*   المجموعة:
    *   إنشاء: إنشاء مجموعة جديدة.
    *   حذف: حذف مجموعة موجودة.
    *   استرداد: استرداد تفاصيل حول مجموعة موجودة.
    *   استرداد متعدد: استرداد قائمة بالمجموعات.
    *   تحديث: تحديث مجموعة موجودة.
*   المستخدم:
    *   إضافة إلى مجموعة: إضافة مستخدم موجود إلى مجموعة.
    *   إنشاء: إنشاء مستخدم جديد.
    *   حذف: حذف مستخدم.
    *   استرداد: استرداد معلومات حول مستخدم موجود.
    *   استرداد متعدد: استرداد قائمة بالمستخدمين.
    *   إزالة من مجموعة: إزالة مستخدم من مجموعة.
    *   تحديث: تحديث مستخدم موجود.
*   مجمع المستخدمين:
    *   استرداد: استرداد معلومات حول مجمع مستخدمين موجود.

## ماذا تفعل إذا كانت عمليتك غير مدعومة

إذا كانت هذه العقدة لا تدعم العملية التي ترغب في تنفيذها، يمكنك استخدام [عقدة HTTP Request](/integrations/builtin/core-nodes/n8n-nodes-base.httprequest/index.md) لاستدعاء واجهة برمجة تطبيقات الخدمة (API).

يمكنك استخدام بيانات الاعتماد التي أنشأتها لهذه الخدمة في عقدة HTTP Request:

1.  في عقدة HTTP Request، حدد **Authentication** > **Predefined Credential Type**.
2.  حدد الخدمة التي ترغب في الاتصال بها.
3.  حدد بيانات الاعتماد الخاصة بك.

ارجع إلى [عمليات API المخصصة](/integrations/custom-operations.md) لمزيد من المعلومات.
