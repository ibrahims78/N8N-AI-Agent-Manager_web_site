# عقدة Segment

استخدم عقدة Segment لأتمتة سير العمل في Segment، ودمج Segment مع تطبيقات أخرى. تدعم n8n بشكل مدمج مجموعة واسعة من ميزات Segment، بما في ذلك إضافة المستخدمين إلى المجموعات، وإنشاء الهويات، وتتبع الأنشطة.

في هذه الصفحة، ستجد قائمة بالعمليات التي تدعمها عقدة Segment وروابط لموارد إضافية.

> **بيانات الاعتماد**
>
> ارجع إلى [بيانات اعتماد Segment](/integrations/builtin/credentials/segment.md) للحصول على إرشادات حول إعداد المصادقة.

## العمليات

* المجموعة
    * إضافة مستخدم إلى مجموعة
* تحديد الهوية
    * إنشاء هوية
* التتبع
    * تسجيل الإجراءات التي يقوم بها المستخدمون. يُحفِّز كل إجراء حدثًا، والذي يمكن أن يحتوي أيضًا على خصائص مرتبطة.
    * تسجيل مشاهدات الصفحات على موقعك الإلكتروني، بالإضافة إلى معلومات إضافية اختيارية حول الصفحة التي يتم عرضها.

## ماذا تفعل إذا كانت عمليتك غير مدعومة

إذا كانت هذه العقدة لا تدعم العملية التي ترغب في تنفيذها، يمكنك استخدام [عقدة HTTP Request (طلب HTTP)](/integrations/builtin/core-nodes/n8n-nodes-base.httprequest/index.md) لاستدعاء واجهة برمجة تطبيقات الخدمة.

يمكنك استخدام بيانات الاعتماد التي أنشأتها لهذه الخدمة في عقدة HTTP Request:

1. في عقدة HTTP Request، حدد **المصادقة** > **نوع بيانات الاعتماد المُحددة مسبقًا**.
2. حدد الخدمة التي ترغب في الاتصال بها.
3. حدد بيانات الاعتماد الخاصة بك.

ارجع إلى [عمليات API المخصصة](/integrations/custom-operations.md) لمزيد من المعلومات.
