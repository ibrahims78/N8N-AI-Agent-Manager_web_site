# HTML

توفر عقدة HTML عمليات لمساعدتك في التعامل مع HTML في n8n.

> **عقدة HTML Extract**
>
> تحل عقدة HTML محل عقدة HTML Extract بدءًا من الإصدار 0.213.0. إذا كنت تستخدم إصدارًا أقدم من n8n، فلا يزال بإمكانك عرض [توثيق عقدة HTML Extract](https://github.com/n8n-io/n8n-docs/blob/86fe33b681621e618e3adcab9a27e8605dbc23ad/docs/integrations/builtin/core-nodes/n8n-nodes-base.htmlextract.md).
> **البرمجة النصية عبر المواقع (Cross-site scripting)**
>
> عند استخدام عقدة HTML لإنشاء قالب HTML، يمكنك إدخال [XSS (البرمجة النصية عبر المواقع)](https://owasp.org/www-community/attacks/xss/). هذه مخاطرة أمنية. كن حذرًا مع المدخلات غير الموثوقة.

## العمليات

* [**إنشاء قالب HTML**](#generate-html-template): استخدم هذه العملية لإنشاء قالب HTML. يتيح لك ذلك أخذ البيانات من سير العمل الخاص بك وإخراجها كـ HTML.
* [**استخراج محتوى HTML**](#extract-html-content): استخراج المحتويات من مصدر بتنسيق HTML. يمكن أن يكون المصدر في JSON أو ملف ثنائي (`.html`).
* [**التحويل إلى جدول HTML**](#convert-to-html-table): تحويل المحتوى إلى جدول HTML.

تعتمد معاملات العقدة والخيارات على العملية التي تحددها. ارجع إلى الأقسام أدناه لمزيد من التفاصيل حول تهيئة كل عملية.

## إنشاء قالب HTML

أنشئ قالب HTML. يتيح لك ذلك أخذ البيانات من سير العمل الخاص بك وإخراجها كـ HTML.

يمكنك تضمين:

* HTML قياسي
* CSS ضمن وسوم `<style>`.
* جافاسكريبت ضمن وسوم `<script>`. لا تقوم n8n بتنفيذ الجافاسكريبت.
* التعبيرات، مغلفة بـ ``.

يمكنك استخدام [التعبيرات](/data/expressions.md) في القالب، بما في ذلك [الدوال والمتغيرات المدمجة](/code/builtin/overview.md) الخاصة بـ n8n.

## استخراج محتوى HTML

استخرج المحتويات من مصدر بتنسيق HTML. يمكن أن يكون المصدر في JSON أو ملف ثنائي (`.html`).

استخدم هذه المعاملات:

### بيانات المصدر

حدد نوع المصدر لمحتوى HTML الخاص بك. اختر بين:

* **JSON**: إذا حددت بيانات المصدر هذه، أدخل **JSON Property**: اسم المدخل الذي يحتوي على HTML الذي تريد استخراجه. يمكن أن تحتوي الخاصية على سلسلة نصية أو مصفوفة من السلاسل النصية.
* **Binary**: إذا حددت بيانات المصدر هذه، أدخل **Input Binary Field**: اسم المدخل الذي يحتوي على HTML الذي تريد استخراجه. يمكن أن تحتوي الخاصية على سلسلة نصية أو مصفوفة من السلاسل النصية.

### قيم الاستخراج

-   **Key**: أدخل المفتاح لحفظ القيمة المستخرجة تحته.
-   **CSS Selector**: أدخل مُحدِّد CSS للبحث عنه.
-   **Return Value**: حدد نوع البيانات المراد إرجاعها. اختر من:
    -   **Attribute**: إرجاع قيمة سمة مثل `class` من عنصر.
        -   إذا اخترت هذا الخيار، أدخل اسم **السمة** لإرجاع قيمتها.
    -   **HTML**: إرجاع محتوى HTML الذي يحتويه العنصر.
    -   **Text**: إرجاع المحتوى النصي للعنصر.
        -   إذا اخترت هذا الخيار، يمكنك أيضًا إدخال قائمة مُحدِّدات مفصولة بفاصلة لتخطيها في **Skip Selectors**.
    -   **Value**: إرجاع قيمة حقل إدخال، أو قائمة اختيار، أو منطقة نصية.
-   **Return Array**: اختر ما إذا كنت تريد إرجاع قيم استخراج متعددة كمصفوفة (مفعل) أو كسلسلة نصية واحدة (معطل).

### خيارات استخراج محتوى HTML

يمكنك أيضًا تهيئة هذه العملية (Operation) باستخدام هذه الخيارات:

*   **Trim Values**: يتحكم فيما إذا كان سيتم إزالة جميع المسافات والأسطر الجديدة من بداية ونهاية القيم (مفعل) أو تركها كما هي (معطل).
*   **Clean Up Text**: يتحكم فيما إذا كان سيتم إزالة المسافات البادئة، والمسافات اللاحقة، وفواصل الأسطر (الأسطر الجديدة) وتكثيف المسافات البيضاء المتعددة المتتالية في مسافة واحدة (مفعل) أو تركها كما هي (معطل).

## تحويل إلى جدول HTML

تتوقع هذه العملية (Operation) بيانات من عقدة (Node) أخرى. ليس لديها أي معاملات (Parameters). تتضمن هذه الخيارات:

*   **Capitalize Headers**: يتحكم فيما إذا كان سيتم تكبير أحرف رؤوس الجدول (مفعل) أم لا (معطل).
*   **Custom Styling**: يتحكم فيما إذا كان سيتم استخدام تنسيق مخصص (مفعل) أم لا (معطل).
*   **Caption**: أدخل تسمية توضيحية لإضافتها إلى الجدول.
*   **Table Attributes**: أدخل أي سمات لتطبيقها على `<table>`، مثل سمات التنسيق.
*   **Header Attributes**: أدخل أي سمات لتطبيقها على رؤوس الجدول `<th>`.
*   **Row Attributes**: أدخل أي سمات لتطبيقها على صفوف الجدول `<tr>`.
*   **Cell Attributes**: أدخل أي سمات لتطبيقها على خلايا الجدول `<td>`.
