# عقدة Microsoft Teams

استخدم عقدة Microsoft Teams لأتمتة سير العمل في Microsoft Teams، ودمج Microsoft Teams مع تطبيقات أخرى. تدعم n8n بشكل مدمج مجموعة واسعة من ميزات Microsoft Teams، بما في ذلك إنشاء وحذف القنوات والرسائل والمهام.

في هذه الصفحة، ستجد قائمة بالعمليات التي تدعمها عقدة Microsoft Teams وروابط لموارد إضافية.

> **بيانات الاعتماد**
>
> ارجع إلى [بيانات اعتماد Microsoft](/integrations/builtin/credentials/microsoft.md) للحصول على إرشادات حول إعداد المصادقة.

> **دعم السحابة الحكومية**
>
> إذا كنت تستخدم مستأجر سحابة حكومية (حكومة الولايات المتحدة، وزارة الدفاع الأمريكية، أو الصين)، فتأكد من تحديد **عنوان URL الأساسي لواجهة برمجة تطبيقات Microsoft Graph** المناسب في إعدادات بيانات اعتماد Microsoft الخاصة بك.

> **يمكن استخدام هذه العقدة كأداة ذكاء اصطناعي**
>
> يمكن استخدام هذه العقدة لتعزيز قدرات وكيل الذكاء الاصطناعي. عند استخدامها بهذه الطريقة، يمكن تعيين العديد من المعاملات تلقائيًا، أو بمعلومات موجهة بواسطة الذكاء الاصطناعي - اكتشف المزيد في [وثائق معاملات أداة الذكاء الاصطناعي](/advanced-ai/examples/using-the-fromai-function.md).

> **التدخل البشري لمكالمات أداة الذكاء الاصطناعي**
>
> يمكن استخدام هذه العقدة كخطوة مراجعة بشرية لمكالمات أداة وكيل الذكاء الاصطناعي. عند تكوينها بهذه الطريقة، سيتوقف وكيل الذكاء الاصطناعي مؤقتًا ويطلب موافقة بشرية عبر هذه الخدمة قبل تنفيذ الأدوات التي تتطلب إشرافًا. تعرف على المزيد في [التدخل البشري لمكالمات أداة الذكاء الاصطناعي](/advanced-ai/human-in-the-loop-tools.md).

## انتظار الاستجابة

باختيار عملية **إرسال وانتظار الاستجابة**، يمكنك إرسال رسالة وإيقاف تنفيذ سير العمل مؤقتًا حتى يؤكد شخص ما الإجراء أو يقدم المزيد من المعلومات.

### نوع الاستجابة

يمكنك الاختيار بين الأنواع التالية من إجراءات الانتظار والموافقة:

*   **موافقة**: يمكن للمستخدمين الموافقة أو الرفض من داخل الرسالة.
*   **نص حر**: يمكن للمستخدمين إرسال استجابة باستخدام نموذج.
*   **نموذج مخصص**: يمكن للمستخدمين إرسال استجابة باستخدام نموذج مخصص.

يمكنك تخصيص سلوك الانتظار والاستجابة بناءً على نوع الاستجابة الذي تختاره. يمكنك تكوين هذه الخيارات في أي من أنواع الاستجابة المذكورة أعلاه:

*   **تحديد وقت الانتظار**: ما إذا كان سير العمل سيستأنف التنفيذ تلقائيًا بعد مهلة زمنية محددة. يمكن أن يكون هذا فاصلًا زمنيًا أو وقتًا محددًا.
*   **إلحاق إسناد n8n**: ما إذا كان سيتم ذكر في الرسالة أنها أُرسلت تلقائيًا باستخدام n8n (مفعل) أم لا (معطل).

### تخصيص استجابة الموافقة

عند استخدام نوع استجابة الموافقة، يمكنك اختيار ما إذا كنت تريد تقديم زر موافقة فقط أو زري موافقة *ورفض*.

يمكنك أيضًا تخصيص تسميات الأزرار للأزرار التي تقوم بتضمينها.

### تخصيص استجابة النص الحر

عند استخدام نوع استجابة النص الحر، يمكنك تخصيص تسمية زر الرسالة، وعنوان النموذج ووصفه، وتسمية زر الاستجابة.

### تخصيص استجابة النموذج المخصص

عند استخدام نوع استجابة النموذج المخصص، تقوم بإنشاء نموذج باستخدام الحقول والخيارات التي تريدها.

يمكنك تخصيص كل عنصر نموذج باستخدام الإعدادات الموضحة في [عناصر نموذج مُحفِّز n8n Form](/integrations/builtin/core-nodes/n8n-nodes-base.formtrigger.md#form-elements). لإضافة المزيد من الحقول، حدد زر **إضافة عنصر نموذج**.

ستتمكن أيضًا من تخصيص تسمية زر الرسالة، وعنوان النموذج ووصفه، وتسمية زر الاستجابة.

## الموارد ذات الصلة

ارجع إلى [توثيق API الخاص بـ Microsoft Teams](https://learn.microsoft.com/en-us/graph/api/overview?view=graph-rest-1.0) لمزيد من المعلومات حول الخدمة.

## ماذا تفعل إذا كانت عمليتك غير مدعومة

إذا كانت هذه العقدة لا تدعم العملية التي تريد القيام بها، يمكنك استخدام [عقدة HTTP Request](/integrations/builtin/core-nodes/n8n-nodes-base.httprequest/index.md) لاستدعاء API الخاص بالخدمة.

يمكنك استخدام بيانات الاعتماد التي أنشأتها لهذه الخدمة في عقدة HTTP Request:

1.  في عقدة HTTP Request، حدد **المصادقة** > **نوع بيانات الاعتماد المُحددة مسبقًا**.
2.  حدد الخدمة التي تريد الاتصال بها.
3.  حدد بيانات الاعتماد الخاصة بك.

ارجع إلى [عمليات API المخصصة](/integrations/custom-operations.md) لمزيد من المعلومات.
