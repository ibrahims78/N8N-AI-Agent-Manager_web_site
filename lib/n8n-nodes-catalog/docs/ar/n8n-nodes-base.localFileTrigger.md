# عقدة Local File Trigger

تبدأ عقدة Local File Trigger سير عمل عندما تكتشف تغييرات في نظام الملفات. تتضمن هذه التغييرات إضافة ملف أو مجلد، أو تعديله، أو حذفه.

> **اعتبارات أمنية**
>
> يمكن أن تُدخل عقدة Local File Trigger مخاطر أمنية كبيرة في البيئات التي تعمل مع مستخدمين غير موثوق بهم. لهذا السبب، يتم [تعطيل](/hosting/securing/blocking-nodes.md#exclude-nodes) العقدة افتراضياً بدءاً من الإصدار 2.0.

> **لنسخ n8n المستضافة ذاتياً فقط**
>
> هذه العقدة غير متاحة على n8n Cloud.

## معاملات العقدة

يمكنك اختيار الحدث المراد مراقبته باستخدام المعامل **Trigger On**.

## التغييرات على ملف محدد

تُحفَّز العقدة عند تغيير الملف المحدد.

أدخل المسار للملف المراد مراقبته في **File to Watch**.

## التغييرات التي تتضمن مجلداً محدداً

تُحفَّز العقدة عند حدوث تغيير في المجلد المحدد.

قم بتكوين هذه المعاملات:

- **Folder to Watch**: أدخل مسار المجلد المراد مراقبته.
- **Watch for**: حدد نوع التغيير المراد مراقبته.

## خيارات العقدة

استخدم **خيارات** العقدة لتضمين أو استبعاد الملفات والمجلدات.

- **Include Linked Files/Folders**: راقب أيضاً التغييرات على الملفات أو المجلدات المرتبطة.
- **Ignore**: الملفات أو المسارات المراد تجاهلها. يختبر n8n المسار كاملاً، وليس اسم الملف فقط. يدعم صيغة [Anymatch](https://github.com/micromatch/anymatch).
- **Max Folder Depth**: مدى العمق في بنية المجلدات لمراقبة التغييرات.

### أمثلة على التجاهل

تجاهل ملفاً واحداً:

```sh
**/<fileName>.<suffix>
# على سبيل المثال، **/myfile.txt
```

تجاهل دليلاً فرعياً ضمن دليل تقوم بمراقبته:

```sh
**/<directoryName>/**
# على سبيل المثال، **/myDirectory/**
```
