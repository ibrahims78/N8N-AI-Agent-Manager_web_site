# عقدة مُحفِّز نموذج n8n

استخدم مُحفِّز نموذج n8n لبدء سير عمل عندما يرسل مستخدم نموذجًا، مع أخذ بيانات المدخلات من النموذج. تُنشئ العقدة صفحة الويب الخاصة بالنموذج لتستخدمها.

يمكنك إضافة المزيد من الصفحات لمتابعة النموذج باستخدام عقدة [n8n Form](/integrations/builtin/core-nodes/n8n-nodes-base.form.md).

## بناء واختبار سير العمل

أثناء بناء أو اختبار سير عمل، استخدم **عنوان URL للاختبار**. يضمن استخدام عنوان URL للاختبار إمكانية عرض البيانات الواردة في واجهة المستخدم للمحرر، وهو مفيد لتصحيح الأخطاء.

هناك طريقتان للاختبار:

- حدد **تنفيذ الخطوة**. يفتح n8n النموذج. عند إرسال النموذج، يقوم n8n بتشغيل العقدة، ولكن ليس بقية سير العمل.
- حدد **تنفيذ سير العمل**. يفتح n8n النموذج. عند إرسال النموذج، يقوم n8n بتشغيل سير العمل.

## سير عمل الإنتاج

عندما يكون سير العمل جاهزًا، انتقل إلى استخدام **عنوان URL للإنتاج**. يمكنك بعد ذلك نشر سير العمل الخاص بك، ويقوم n8n بتشغيله تلقائيًا عندما يرسل مستخدم النموذج.

عند العمل باستخدام عنوان URL للإنتاج، تأكد من حفظ ونشر سير العمل. البيانات المتدفقة عبر مُحفِّز النموذج لا تكون مرئية في واجهة المستخدم للمحرر عند استخدام عنوان URL للإنتاج.

## تعيين التحديدات الافتراضية باستخدام معاملات الاستعلام

يمكنك تعيين القيم الأولية للحقول باستخدام [معاملات الاستعلام](https://en.wikipedia.org/wiki/Query_string#Web_forms) مع عنوان URL الأولي الذي يوفره مُحفِّز نموذج n8n. تستقبل كل [صفحة في النموذج](/integrations/builtin/core-nodes/n8n-nodes-base.form.md) نفس معاملات الاستعلام المرسلة إلى عنوان URL الخاص بمُحفِّز نموذج n8n.

> **للإنتاج فقط**
>
> تتوفر معاملات الاستعلام فقط عند استخدام النموذج في وضع الإنتاج. لن يقوم n8n بتعبئة قيم الحقول من معاملات الاستعلام في وضع الاختبار.

عند استخدام معاملات الاستعلام، قم [بترميز النسبة المئوية (percent-encode)](https://en.wikipedia.org/wiki/Percent-encoding) لأي أسماء حقول أو قيم تستخدم أحرفًا خاصة. يضمن هذا أن n8n يستخدم القيم الأولية للحقول المعطاة. يمكنك استخدام أدوات مثل [URL Encode/Decode](https://www.url-encode-decode.com/) لتنسيق معاملات الاستعلام الخاصة بك باستخدام ترميز النسبة المئوية.

على سبيل المثال، تخيل أن لديك نموذجًا بالخصائص التالية:

* عنوان URL للإنتاج: `https://my-account.n8n.cloud/form/my-form`
* الحقول:
	* `name`: `Jane Doe`
	* `email`: `jane.doe@example.com`

باستخدام معاملات الاستعلام وترميز النسبة المئوية، يمكنك استخدام عنوان URL التالي لتعيين قيم الحقول الأولية للبيانات المذكورة أعلاه:

```
https://my-account.n8n.cloud/form/my-form?email=jane.doe%40example.com&name=Jane%20Doe
```

هنا، يستبدل ترميز النسبة المئوية الرمز @ (`@`) بالسلسلة `%40` وحرف المسافة (` `) بالسلسلة `%20`. سيؤدي هذا إلى تعيين القيمة الأولية لهذه الحقول بغض النظر عن الصفحة التي تظهر فيها في النموذج.

```markdown
## معاملات العقدة

هذه هي حقول تهيئة العقدة الرئيسية:

### المصادقة

- **مصادقة أساسية (Basic Auth)**
- **بلا (None)**

#### استخدام المصادقة الأساسية

لتهيئة بيانات الاعتماد هذه، ستحتاج إلى:

- **اسم المستخدم (Username)** الذي تستخدمه للوصول إلى التطبيق أو الخدمة التي يستهدفها طلب HTTP الخاص بك.
- **كلمة المرور (Password)** المرتبطة باسم المستخدم هذا.

### عناوين URL للنماذج

تحتوي عقدة مُحفِّز النموذج (Form Trigger) على عنواني URL: **عنوان URL للاختبار (Test URL)** و**عنوان URL للإنتاج (Production URL)**. تعرض n8n عناوين URL في الجزء العلوي من لوحة العقدة. حدد **عنوان URL للاختبار** أو **عنوان URL للإنتاج** للتبديل بين عنوان URL الذي تعرضه n8n.

![لقطة شاشة لعناوين URL للنماذج](/_images/integrations/builtin/core-nodes/form-trigger/form-urls.png)

- **عنوان URL للاختبار**: تسجل n8n ويب هوك اختباريًا عند تحديد **تنفيذ الخطوة (Execute Step)** أو **تنفيذ سير العمل (Execute Workflow)**، إذا لم يكن سير العمل نشطًا. عند استدعاء عنوان URL، تعرض n8n البيانات في سير العمل.
- **عنوان URL للإنتاج**: تسجل n8n ويب هوك إنتاجيًا عند نشر سير العمل. عند استخدام عنوان URL للإنتاج، لا تعرض n8n البيانات في سير العمل. لا يزال بإمكانك عرض بيانات سير العمل لتنفيذ إنتاجي. حدد علامة التبويب **التنفيذات (Executions)** في سير العمل، ثم حدد تنفيذ سير العمل الذي تريد عرضه.

### مسار النموذج

عيّن مسارًا مخصصًا (slug) للنموذج.

### عنوان النموذج

أدخل عنوان النموذج الخاص بك. تعرض n8n **عنوان النموذج** كعنوان لصفحة الويب وعنوان `h1` الرئيسي في النموذج.

### وصف النموذج

أدخل وصف النموذج الخاص بك. تعرض n8n **وصف النموذج** كعنوان فرعي أسفل عنوان `h1` الرئيسي في النموذج. استخدم `\n` أو `<br>` لإضافة فاصل أسطر.

للحصول على معلومات حول علامات HTML المسموح بها والمقيدة، راجع [أمان HTML والعلامات المسموح بها](#html-security-and-allowed-tags).

### عناصر النموذج

أنشئ حقول الأسئلة لنموذجك. حدد **إضافة عنصر نموذج (Add Form Element)** لإضافة حقل جديد.

يحتوي كل حقل على الإعدادات التالية:

- **تسمية الحقل (Field Label)**: أدخل التسمية التي تظهر فوق حقل الإدخال في النموذج المعروض.
- **اسم الحقل (Field Name)**: يُستخدم هذا الاسم في مخرجات عقدة مُحفِّز النموذج. استخدمه للإشارة إلى حقل نموذج في العقد اللاحقة.
- **نوع العنصر (Element Type)**: اختر من بين **مربعات الاختيار (Checkboxes)**، **HTML مخصص (Custom HTML)**، **تاريخ (Date)**، **قائمة منسدلة (Dropdown)**، **بريد إلكتروني (Email)**، **ملف (File)**، **حقل مخفي (Hidden Field)**، **رقم (Number)**، **كلمة مرور (Password)**، **أزرار اختيار (Radio Buttons)**، **نص (Text)**، أو **مربع نص (Textarea)**.
    - حدد **مربعات الاختيار** لتضمين عناصر مربعات الاختيار في النموذج. بشكل افتراضي، لا يوجد حد لعدد مربعات الاختيار التي يمكن لمستخدم النموذج تحديدها. يمكنك تعيين الحد عن طريق تحديد قيمة لخيار **تحديد الحد (Limit Selection)** كـ **عدد دقيق (Exact Number)**، **نطاق (Range)**، أو **غير محدود (Unlimited)**.
    - حدد **HTML مخصص** لإدراج HTML عشوائي.
        - يمكنك تضمين عناصر مثل الروابط والصور والفيديو والمزيد. لا يمكنك تضمين عناصر `<script>` أو `<style>` أو `<input>`. لمزيد من المعلومات، راجع [أمان HTML والعلامات المسموح بها](#html-security-and-allowed-tags).
        - بشكل افتراضي، لا يتم تضمين حقول HTML المخصصة في مخرجات العقدة. لتضمين محتوى HTML المخصص في المخرجات، املأ حقل **اسم العنصر (Element Name)** المرتبط.
    - حدد **تاريخ** لتضمين منتقي تاريخ في النموذج. ارجع إلى [التاريخ والوقت مع Luxon](/data/specific-data-types/luxon.md) لمزيد من المعلومات حول تنسيق التواريخ.
    - حدد **قائمة منسدلة** > **إضافة خيار حقل (Add Field Option)** لإضافة خيارات متعددة. بشكل افتراضي، تكون القائمة المنسدلة ذات اختيار واحد. لجعلها متعددة الاختيارات، قم بتشغيل **اختيار متعدد (Multiple Choice)**.
    - حدد **أزرار اختيار** لتضمين عناصر أزرار الاختيار في النموذج.
    - حدد **حقل مخفي** لتضمين عنصر نموذج دون عرضه في النموذج. يمكنك تعيين قيمة افتراضية باستخدام المعامل **قيمة الحقل (Field Value)** أو تمرير قيم للحقل باستخدام [معاملات الاستعلام](#set-default-selections-with-query-parameters).
```

- **عنصر نائب**: يُعرّف نصًا نموذجيًا ليُعرض داخل عناصر النموذج المتوافقة. يتم دعم العناصر النائبة في حقول **البريد الإلكتروني**، **الرقم**، **كلمة المرور**، **النص**، و**منطقة النص**.
- **القيمة الافتراضية**: تُعرّف قيمة افتراضية يتم ملؤها مسبقًا أو تحديدها مسبقًا في عناصر النموذج المتوافقة. يتم دعم القيم الافتراضية في جميع عناصر النموذج باستثناء **HTML مخصص**، **الملف**، **الحقل المخفي**، و**كلمة المرور**.
- **حقل مطلوب**: قم بتشغيله لمطالبة المستخدمين بإكمال هذا الحقل في النموذج.

### الاستجابة عند

اختر متى ترسل n8n استجابة لإرسال النموذج. يمكنك الاستجابة عندما:

- **تم إرسال النموذج**: أرسل استجابة للمستخدم بمجرد إرساله للنموذج.
- **ينتهي سير العمل**: استخدم هذا إذا كنت تريد أن يكمل سير العمل تنفيذه قبل إرسال استجابة للمستخدم. إذا حدث خطأ في سير العمل، فإنه يرسل استجابة للمستخدم لإخباره بوجود مشكلة في إرسال النموذج.

## خيارات العقدة

حدد **إضافة خيار** لعرض المزيد من خيارات التهيئة:

- **إلحاق إسناد n8n**: قم بإيقاف تشغيله لإخفاء سمة **النموذج مؤتمت بواسطة n8n** في أسفل النموذج.
- **تسمية الزر**: التسمية التي ستُستخدم لزر إرسال النموذج الخاص بك. تعرض n8n **تسمية الزر** كاسم لزر الإرسال.
- **مسار النموذج**: الجزء الأخير من عنوان URL للنموذج، لكل من الاختبار والإنتاج. يحل محل UUID الذي تم إنشاؤه تلقائيًا كمكون نهائي.
- **تجاهل الروبوتات**: قم بتشغيله لتجاهل الطلبات من الروبوتات مثل معاينات الروابط وزواحف الويب.
- **استخدام المنطقة الزمنية لسير العمل**: قم بتشغيله لاستخدام المنطقة الزمنية في [إعدادات سير العمل](/workflows/settings.md) بدلاً من التوقيت العالمي المنسق (UTC) (الافتراضي). يؤثر هذا على قيمة الطابع الزمني `submittedAt` في مخرجات العقدة.
- **تخصيص نمط النموذج**: تجاوز النمط الافتراضي لواجهة النموذج العامة باستخدام CSS. يتم ملء الحقل مسبقًا بالنمط الافتراضي حتى تتمكن من تغيير ما تحتاجه فقط.

## تخصيص سلوك عقدة مُحفِّز النموذج

### تنسيق نص الاستجابة بفاصل الأسطر

يمكنك استخدام إحدى الطرق التالية لإضافة فواصل أسطر إلى نص استجابة النموذج:

• استخدم تنسيق HTML بدلاً من النص العادي في حقل `formSubmittedText`
• استبدل أحرف السطر الجديد (`\n`) بعلامات فواصل HTML (`<br>`) قبل إرسال الاستجابة
• فكر في استخدام صفحة استجابة HTML مخصصة إذا كنت بحاجة إلى مزيد من التحكم في التنسيق

### تقييد الوصول إلى النموذج باستخدام المصادقة

يمكنك استخدام أحد الخيارات التالية لإضافة المصادقة إلى نموذجك:

*   استخدام حقل OTP (كلمة المرور لمرة واحدة) مع التحقق من صحة العقدة (Node) TOTP للمصادقة القائمة على الرمز المميز.
*   إضافة عقدة Wait مع مصادقة النموذج كصفحة نموذج ثانوية.
*   تخزين كلمات المرور المُجزأة في قاعدة بيانات ومقارنتها بإرسالات النموذج للتحقق من الصحة.
*   استخدام موفري مصادقة خارجيين مثل Google Forms إذا كنت بحاجة إلى مصادقة متقدمة.

## أمان HTML والعلامات المسموح بها

يقوم مُحفِّز (Trigger) النموذج في n8n تلقائيًا بتنقية محتوى HTML في حقل **وصف النموذج** وعناصر النموذج **HTML المخصص** لمنع الثغرات الأمنية. بينما يتم دعم HTML للتنسيق، فإن بعض العلامات والسمات مقيدة.

### علامات HTML المسموح بها

يمكنك استخدام العلامات التالية للتنسيق: `<a>`, `<b>`, `<br>`, `<code>`, `<div>`, `<em>`, `<h1>` حتى `<h6>`, `<i>`, `<iframe>`, `<img>`, `<li>`, `<ol>`, `<p>`, `<pre>`, `<span>`, `<strong>`, `<sub>`, `<sup>`, `<table>`, `<tbody>`, `<td>`, `<tfoot>`, `<th>`, `<thead>`, `<tr>`, `<u>`, `<ul>`, `<video>`, و `<source>`.

### العلامات المحظورة

تتم إزالة العلامات التالية تلقائيًا لأسباب أمنية: `<script>`, `<style>`, `<input>`, `<form>`, `<button>`، وعناصر أخرى قد تكون خطيرة ويمكن أن تمكّن هجمات XSS أو تتداخل مع وظائف النموذج.

### قيود السمات

يُسمح فقط بسمات محددة على علامات معينة:

*   الروابط (<a>): `href`, `target`, `rel`
*   الصور (<img>): `src`, `alt`, `width`, `height`
*   مقاطع الفيديو (<video>): `controls`, `autoplay`, `loop`, `muted`, `poster`, `width`, `height`
*   الإطارات المتداخلة (<iframe>): `src`, `width`, `height`, `frameborder`, `allow`, `allowfullscreen`, `referrerpolicy` (يتم وضعها في بيئة معزولة تلقائيًا)
*   خلايا الجدول (<td>, <th>): `colspan`, `rowspan`, `scope`, `headers`

تتم إزالة جميع السمات الأخرى أثناء التنقية. يُسمح فقط بمخططات URL التالية: `http://` و `https://`.
