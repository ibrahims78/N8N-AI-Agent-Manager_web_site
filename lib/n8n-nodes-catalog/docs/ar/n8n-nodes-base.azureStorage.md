# عقدة Azure Storage

تدعم عقدة Azure Storage (تخزين أزور) بشكل مدمج مجموعة واسعة من الميزات، والتي تشمل إنشاء كائنات التخزين الثنائية (blobs) والحاويات واستردادها وحذفها. استخدم هذه العقدة لأتمتة العمل ضمن خدمة Azure Storage أو لدمجها مع خدمات أخرى في سير العمل الخاص بك.

في هذه الصفحة، ستجد قائمة بالعمليات التي تدعمها عقدة Azure Storage، وروابط لموارد إضافية.

> **بيانات الاعتماد**
>
> يمكنك العثور على معلومات المصادقة لهذه العقدة [هنا](/integrations/builtin/credentials/azurestorage.md).

## العمليات

*   **كائن التخزين الثنائي (Blob)**
    *   **إنشاء كائن تخزين ثنائي (blob)**: إنشاء كائن تخزين ثنائي جديد أو استبدال كائن موجود.
    *   **حذف كائن تخزين ثنائي (blob)**: حذف كائن تخزين ثنائي موجود.
    *   **استرداد كائن تخزين ثنائي (blob)**: استرداد البيانات لكائن تخزين ثنائي محدد.
    *   **استرداد العديد من كائنات التخزين الثنائية (blobs)**: استرداد قائمة بكائنات التخزين الثنائية.
*   **الحاوية (Container)**
    *   **إنشاء حاوية**: إنشاء حاوية جديدة.
    *   **حذف حاوية**: حذف حاوية موجودة.
    *   **استرداد حاوية**: استرداد البيانات لحاوية محددة.
    *   **استرداد العديد من الحاويات**: استرداد قائمة بالحاويات.

## ماذا تفعل إذا كانت العملية المطلوبة غير مدعومة

إذا كانت هذه العقدة لا تدعم العملية التي ترغب في القيام بها، يمكنك استخدام [عقدة HTTP Request](/integrations/builtin/core-nodes/n8n-nodes-base.httprequest/index.md) لاستدعاء واجهة برمجة تطبيقات (API) الخدمة.

يمكنك استخدام بيانات الاعتماد التي أنشأتها لهذه الخدمة في عقدة HTTP Request:

1.  في عقدة HTTP Request، حدد **Authentication** > **Predefined Credential Type**.
2.  حدد الخدمة التي تريد الاتصال بها.
3.  حدد بيانات الاعتماد الخاصة بك.

ارجع إلى [عمليات API المخصصة](/integrations/custom-operations.md) لمزيد من المعلومات.
