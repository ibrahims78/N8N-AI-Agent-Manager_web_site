# عقدة Microsoft Outlook

استخدم عقدة Microsoft Outlook لأتمتة العمل في Microsoft Outlook، ودمج Microsoft Outlook مع تطبيقات أخرى. يدعم n8n مجموعة واسعة من ميزات Microsoft Outlook بشكل مدمج، بما في ذلك إنشاء المجلدات والرسائل والمسودات وتحديثها وحذفها وجلبها.

ستجد في هذه الصفحة قائمة بالعمليات التي تدعمها عقدة Microsoft Outlook وروابط لموارد إضافية.

> **بيانات الاعتماد**
>
> ارجع إلى [بيانات اعتماد Microsoft](/integrations/builtin/credentials/microsoft.md) للحصول على إرشادات حول إعداد المصادقة.

> **دعم السحابة الحكومية**
>
> إذا كنت تستخدم مستأجر سحابة حكومية (حكومة الولايات المتحدة، وزارة الدفاع الأمريكية، أو الصين)، فتأكد من تحديد **Microsoft Graph API Base URL** المناسب في تهيئة بيانات اعتماد Microsoft الخاصة بك.

> **يمكن استخدام هذه العقدة كأداة ذكاء اصطناعي**
>
> يمكن استخدام هذه العقدة لتعزيز قدرات وكيل الذكاء الاصطناعي. عند استخدامها بهذه الطريقة، يمكن تعيين العديد من المعاملات تلقائيًا، أو بمعلومات موجهة بواسطة الذكاء الاصطناعي - اكتشف المزيد في [توثيق معاملات أداة الذكاء الاصطناعي](/advanced-ai/examples/using-the-fromai-function.md).

> **التدخل البشري في استدعاءات أدوات الذكاء الاصطناعي**
>
> يمكن استخدام هذه العقدة كخطوة مراجعة بشرية لاستدعاءات أدوات وكيل الذكاء الاصطناعي. عند تهيئتها بهذه الطريقة، سيتوقف وكيل الذكاء الاصطناعي مؤقتًا ويطلب موافقة بشرية عبر هذه الخدمة قبل تنفيذ الأدوات التي تتطلب إشرافًا. تعرف على المزيد في [التدخل البشري في استدعاءات أدوات الذكاء الاصطناعي](/advanced-ai/human-in-the-loop-tools.md).

## انتظار الرد

باختيار عملية **إرسال وانتظار الرد**، يمكنك إرسال رسالة وإيقاف تنفيذ سير العمل مؤقتًا حتى يؤكد شخص ما الإجراء أو يقدم المزيد من المعلومات.

### نوع الاستجابة

يمكنك الاختيار بين الأنواع التالية من إجراءات الانتظار والموافقة:

*   **الموافقة**: يمكن للمستخدمين الموافقة أو عدم الموافقة من داخل الرسالة.
*   **نص حر**: يمكن للمستخدمين إرسال استجابة باستخدام نموذج.
*   **نموذج مخصص**: يمكن للمستخدمين إرسال استجابة باستخدام نموذج مخصص.

يمكنك تخصيص سلوك الانتظار والاستجابة بناءً على نوع الاستجابة الذي تختاره. يمكنك تهيئة هذه الخيارات في أي من أنواع الاستجابة المذكورة أعلاه:

*   **تحديد مهلة الانتظار**: ما إذا كان سير العمل (Workflow) سيستأنف التنفيذ (Execution) تلقائيًا بعد مهلة زمنية محددة. يمكن أن تكون هذه المهلة فترة زمنية أو وقتًا محددًا.
*   **إلحاق إسناد n8n**: ما إذا كان سيتم ذكر في الرسالة أنها أُرسلت تلقائيًا باستخدام n8n (مفعل) أم لا (معطل).

### تخصيص استجابة الموافقة

عند استخدام نوع استجابة الموافقة، يمكنك اختيار ما إذا كنت تريد عرض زر موافقة فقط أو زرَي موافقة *و* عدم موافقة.

يمكنك أيضًا تخصيص تسميات الأزرار للأزرار التي تقوم بتضمينها.

### تخصيص استجابة النص الحر

عند استخدام نوع استجابة النص الحر، يمكنك تخصيص تسمية زر الرسالة، وعنوان النموذج ووصفه، وتسمية زر الاستجابة.

### تخصيص استجابة النموذج المخصص

عند استخدام نوع استجابة النموذج المخصص، يمكنك بناء نموذج باستخدام الحقول والخيارات التي تريدها.

يمكنك تخصيص كل عنصر (Item) نموذج باستخدام الإعدادات الموضحة في [عناصر نموذج مُحفِّز (Trigger) نموذج n8n](/integrations/builtin/core-nodes/n8n-nodes-base.formtrigger.md#form-elements). لإضافة المزيد من الحقول، حدد زر **إضافة عنصر نموذج**.

ستتمكن أيضًا من تخصيص تسمية زر الرسالة، وعنوان النموذج ووصفه، وتسمية زر الاستجابة.

## الموارد ذات الصلة

ارجع إلى [توثيق API الخاص بـ Outlook](https://learn.microsoft.com/en-us/outlook/rest/get-started) لمزيد من المعلومات حول الخدمة.

## ماذا تفعل إذا كانت عمليتك غير مدعومة

إذا كانت هذه العقدة (Node) لا تدعم العملية (Operation) التي تريد القيام بها، يمكنك استخدام [عقدة (Node) طلب HTTP](/integrations/builtin/core-nodes/n8n-nodes-base.httprequest/index.md) لاستدعاء API الخاص بالخدمة.

يمكنك استخدام بيانات الاعتماد (Credential) التي أنشأتها لهذه الخدمة في عقدة (Node) طلب HTTP:

1.  في عقدة (Node) طلب HTTP، حدد **المصادقة (Authentication)** > **نوع بيانات الاعتماد (Credential) المُحددة مسبقًا**.
2.  حدد الخدمة التي تريد الاتصال (Connection) بها.
3.  حدد بيانات الاعتماد (Credential) الخاصة بك.

ارجع إلى [عمليات API المخصصة](/integrations/custom-operations.md) لمزيد من المعلومات.
