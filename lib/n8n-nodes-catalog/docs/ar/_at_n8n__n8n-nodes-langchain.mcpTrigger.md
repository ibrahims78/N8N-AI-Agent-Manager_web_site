# عقدة مُحفِّز خادم MCP (MCP Server Trigger node)

استخدم عقدة مُحفِّز خادم MCP للسماح لـ n8n بالعمل كخادم [بروتوكول سياق النموذج (MCP)](https://modelcontextprotocol.io/introduction)، مما يجعل أدوات n8n و سير العمل متاحة لعملاء MCP.

> **بيانات الاعتماد**
>
> يمكن العثور على معلومات المصادقة لهذه العقدة [هنا](/integrations/builtin/credentials/httprequest.md).

## كيفية عمل عقدة مُحفِّز خادم MCP

تعمل عقدة مُحفِّز خادم MCP كنقطة دخول إلى n8n لعملاء MCP. تعمل عن طريق كشف عنوان URL يمكن لعملاء MCP التفاعل معه للوصول إلى أدوات n8n.

على عكس [عقد المُحفِّز](/glossary.md#trigger-node-n8n) التقليدية، التي تستجيب للأحداث وتمرر مخرجاتها إلى [العقدة المتصلة](/workflows/components/connections.md) التالية، تتصل عقدة مُحفِّز خادم MCP فقط بعقد [الأداة](/advanced-ai/examples/understand-tools.md) وتنفذها. يمكن للعملاء سرد الأدوات المتاحة واستدعاء أدوات فردية لأداء العمل.

يمكنك كشف سير عمل n8n للعملاء عن طريق إرفاقها بعقدة [أداة سير عمل n8n المخصصة](/integrations/builtin/cluster-nodes/sub-nodes/n8n-nodes-langchain.toolworkflow.md).

> **دعم أحداث الخادم المرسلة (SSE) و HTTP القابل للتدفق**
>
> تدعم عقدة مُحفِّز خادم MCP كلاً من [أحداث الخادم المرسلة (SSE)](https://modelcontextprotocol.io/docs/concepts/transports#server-sent-events-sse)، وهو نقل طويل الأمد مبني فوق HTTP، و [HTTP القابل للتدفق](https://modelcontextprotocol.io/specification/2025-03-26/basic/transports#streamable-http) للاتصالات بين العملاء والخادم. لا تدعم حاليًا نقل [المدخلات/المخرجات القياسية (stdio)](https://modelcontextprotocol.io/docs/concepts/transports#standard-input%2Foutput-stdio).

## معاملات العقدة

استخدم هذه المعاملات لتكوين العقدة الخاصة بك.

### عنوان URL لـ MCP

تحتوي عقدة مُحفِّز خادم MCP على عنواني **URL لـ MCP**: للاختبار والإنتاج. يعرض n8n عناوين URL في الجزء العلوي من لوحة العقدة.

حدد **عنوان URL للاختبار** أو **عنوان URL للإنتاج** للتبديل بين عنواني URL اللذين يعرضهما n8n.

*   **الاختبار**: يسجل n8n عنوان URL لـ MCP للاختبار عند تحديد **الاستماع لحدث الاختبار** أو **تنفيذ سير العمل**، إذا لم يكن سير العمل نشطًا. عند استدعاء عنوان URL لـ MCP، يعرض n8n البيانات في سير العمل.
*   **الإنتاج**: يسجل n8n عنوان URL لـ MCP للإنتاج عند نشر سير العمل. عند استخدام عنوان URL للإنتاج، لا يعرض n8n البيانات في سير العمل. لا يزال بإمكانك عرض بيانات سير العمل لتنفيذ إنتاجي: حدد علامة التبويب **التنفيذات** في سير العمل، ثم حدد تنفيذ سير العمل الذي ترغب في عرضه.

### المصادقة

يمكنك طلب المصادقة للعملاء الذين يتصلون بعنوان URL الخاص بـ MCP. اختر من طرق المصادقة التالية:

-   مصادقة Bearer
-   مصادقة Header

ارجع إلى [بيانات اعتماد طلب HTTP](/integrations/builtin/credentials/httprequest.md) لمزيد من المعلومات حول إعداد كل نوع من أنواع بيانات الاعتماد.

### المسار

بشكل افتراضي، يحتوي هذا الحقل على مسار URL لـ MCP تم إنشاؤه عشوائيًا، لتجنب التعارضات مع عقد مُحفِّز خادم MCP الأخرى.

يمكنك تحديد مسار URL يدويًا، بما في ذلك إضافة مُعاملات المسار. على سبيل المثال، قد تحتاج إلى القيام بذلك إذا كنت تستخدم n8n لإنشاء نموذج أولي لواجهة برمجة تطبيقات (API) وترغب في الحصول على عناوين URL متسقة لنقاط النهاية.

## القوالب والأمثلة

### التكامل مع Claude Desktop

يمكنك الاتصال بعقدة مُحفِّز خادم MCP من [Claude Desktop](https://claude.ai/download) عن طريق تشغيل بوابة لعمل وكيل لرسائل SSE إلى الخوادم المستندة إلى stdio.

للقيام بذلك، أضف ما يلي إلى تهيئة Claude Desktop الخاص بك:

```json
{
  "mcpServers": {
    "n8n": {
      "command": "npx",
      "args": [
        "mcp-remote",
        "<MCP_URL>",
        "--header",
        "Authorization: Bearer ${AUTH_TOKEN}"
      ],
      "env": {
        "AUTH_TOKEN": "<MCP_BEARER_TOKEN>"
      }
    }
  }
}
```

تأكد من استبدال العناصر النائبة `<MCP_URL>` و `<MCP_BEARER_TOKEN>` بالقيم من مُعاملات عقدة مُحفِّز خادم MCP وبيانات الاعتماد الخاصة بك.

## القيود

### تهيئة عقدة مُحفِّز خادم MCP مع نسخ الويب هوك

تعتمد عقدة مُحفِّز خادم MCP على أحداث مرسلة من الخادم (SSE) أو HTTP القابل للتدفق، والتي تتطلب نفس نسخة الخادم للتعامل مع الاتصالات المستمرة. قد يتسبب هذا في مشاكل عند تشغيل n8n في [وضع قائمة الانتظار](/hosting/scaling/queue-mode.md) اعتمادًا على تهيئة [معالج الويب هوك](/hosting/scaling/queue-mode.md#webhook-processors) الخاص بك:

*   إذا كنت تستخدم وضع قائمة الانتظار مع **نسخة ويب هوك واحدة**، فإن عقدة مُحفِّز خادم MCP تعمل كما هو متوقع.
*   إذا كنت تقوم بتشغيل **نسخ ويب هوك متعددة**، فأنت بحاجة إلى توجيه جميع طلبات `/mcp*` إلى نسخة ويب هوك واحدة مخصصة. أنشئ مجموعة نسخ منفصلة تحتوي على حاوية ويب هوك واحدة لطلبات MCP. بعد ذلك، قم بتحديث تهيئة الدخول (ingress) أو موازن التحميل (load balancer) الخاص بك لتوجيه جميع حركة مرور `/mcp*` إلى تلك النسخة.

> **تحذير عند التشغيل مع نسخ ويب هوك متعددة**
>
> إذا قمت بتشغيل عقدة مُحفِّز خادم MCP مع نسخ ويب هوك متعددة ولم تقم بتوجيه جميع طلبات `/mcp*` إلى نسخة ويب هوك واحدة مخصصة، فإن اتصالات SSE و HTTP القابلة للتدفق الخاصة بك ستنقطع بشكل متكرر أو تفشل في تسليم الأحداث بشكل موثوق.

## الموارد ذات الصلة

يوفر n8n أيضًا [عقدة أداة عميل MCP](/integrations/builtin/cluster-nodes/sub-nodes/n8n-nodes-langchain.toolmcp.md) التي تسمح لك بربط وكلاء n8n للذكاء الاصطناعي الخاصين بك بالأدوات الخارجية.

ارجع إلى [توثيق MCP](https://modelcontextprotocol.io/introduction) و[مواصفات MCP](https://modelcontextprotocol.io/specification/) لمزيد من التفاصيل حول البروتوكول والخوادم والعملاء.

## المشكلات الشائعة

فيما يلي بعض الأخطاء والمشكلات الشائعة المتعلقة بعقدة مُحفِّز خادم MCP وخطوات حلها أو استكشاف أخطائها وإصلاحها.

### تشغيل عقدة مُحفِّز خادم MCP مع وكيل عكسي

عند تشغيل n8n خلف وكيل عكسي (reverse proxy) مثل nginx، قد تواجه مشكلات إذا لم يتم تكوين نقطة نهاية MCP لـ SSE أو HTTP القابل للتدفق.

على وجه التحديد، تحتاج إلى تعطيل التخزين المؤقت للوكيل (proxy buffering) لنقطة النهاية. تتضمن العناصر الأخرى التي قد ترغب في تعديلها تعطيل ضغط gzip (يتعامل n8n مع هذا بنفسه)، وتعطيل ترميز النقل المجزأ (chunked transfer encoding)، وتعيين `Connection` إلى سلسلة فارغة لإزالته من الرؤوس المعاد توجيهها. يضمن تعطيل هذه الإعدادات صراحةً في نقطة نهاية MCP عدم وراثتها من أماكن أخرى في تكوين nginx الخاص بك.

قد يبدو مثال على كتلة موقع (location block) في nginx لخدمة حركة مرور MCP بهذه الإعدادات كما يلي:

```
location /mcp/ {
    proxy_http_version          1.1;
    proxy_buffering             off;
    gzip                        off;
    chunked_transfer_encoding   off;

    proxy_set_header            Connection '';

    # The rest of your proxy headers and settings
    # . . .
}
```
