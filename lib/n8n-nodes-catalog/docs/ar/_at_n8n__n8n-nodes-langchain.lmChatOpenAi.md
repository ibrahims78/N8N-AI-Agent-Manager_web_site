# عقدة OpenAI Chat Model

استخدم عقدة OpenAI Chat Model لاستخدام نماذج الدردشة الخاصة بـ OpenAI مع [الوكلاء](/glossary.md#ai-agent) التخاطبيين.

في هذه الصفحة، ستجد معاملات العقدة لعقدة OpenAI Chat Model وروابط لموارد إضافية.

> **بيانات الاعتماد**
>
> يمكنك العثور على معلومات المصادقة لهذه العقدة [هنا](/integrations/builtin/credentials/openai.md).

> **حل المعاملات في العقد الفرعية**
>
> تتصرف العقد الفرعية بشكل مختلف عن العقد الأخرى عند معالجة عناصر متعددة باستخدام تعبير.
>
> تأخذ معظم العقد، بما في ذلك العقد الجذرية، أي عدد من العناصر كمدخلات، وتعالج هذه العناصر، وتُخرج النتائج. يمكنك استخدام التعبيرات للإشارة إلى عناصر المدخلات، وتقوم العقدة بحل التعبير لكل عنصر على حدة. على سبيل المثال، بالنظر إلى مدخلات تحتوي على خمس قيم `name`، يتم حل التعبير `{{ $item.json.name }}` لكل اسم على حدة.
>
> في العقد الفرعية، يتم حل التعبير دائمًا إلى العنصر الأول. على سبيل المثال، بالنظر إلى مدخلات تحتوي على خمس قيم `name`، يتم حل التعبير `{{ $item.json.name }}` دائمًا إلى الاسم الأول.

## معاملات العقدة

### النموذج

حدد النموذج الذي سيتم استخدامه لتوليد الإكمال.

تقوم n8n بتحميل النماذج ديناميكيًا من OpenAI، وستظهر لك فقط النماذج المتاحة لحسابك.

### استخدام Responses API
توفر OpenAI نقطتي نهاية لتوليد المخرجات من نموذج:
- **إكمال الدردشة (Chat Completions)**: تقوم نقطة نهاية Chat Completions API بتوليد استجابة نموذج من قائمة رسائل تشكل محادثة. تتطلب واجهة API من المستخدم التعامل مع حالة المحادثة يدويًا، على سبيل المثال عن طريق إضافة عقدة فرعية [Simple Memory](/integrations/builtin/cluster-nodes/sub-nodes/n8n-nodes-langchain.memorybufferwindow/index.md). للمشاريع الجديدة، توصي OpenAI باستخدام Responses API.
- **الاستجابات (Responses)**: إن Responses API هي حلقة وكيلية، تسمح للنموذج باستدعاء أدوات مدمجة متعددة ضمن نطاق طلب API واحد. كما أنها تدعم المحادثات المستمرة عن طريق تمرير `conversation_id`.

قم بالتبديل إلى **استخدام Responses API** إذا كنت تريد أن يقوم النموذج بتوليد المخرجات باستخدام Responses API. وإلا، فإن عقدة OpenAI Chat Model ستستخدم Chat Completions API افتراضيًا.

ارجع إلى وثائق OpenAI للاطلاع على [مقارنة بين Chat Completions و Responses APIs](https://platform.openai.com/docs/guides/migrate-to-responses).

### الأدوات المدمجة
توفر واجهة برمجة تطبيقات استجابات OpenAI (OpenAI Responses API) مجموعة من [الأدوات المدمجة](https://platform.openai.com/docs/guides/tools) لإثراء استجابة النموذج. قم بالتبديل إلى **استخدام Responses API** إذا كنت تريد أن يتمكن النموذج من الوصول إلى الأدوات المدمجة التالية:

-   **بحث الويب**: يسمح للنماذج بالبحث في الويب عن أحدث المعلومات قبل إنشاء استجابة.
-   **بحث الملفات**: يسمح للنماذج بالبحث في قاعدة معارفك من الملفات التي تم تحميلها مسبقًا عن المعلومات ذات الصلة قبل إنشاء استجابة. ارجع إلى [توثيق OpenAI](https://platform.openai.com/docs/guides/tools-file-search) لمزيد من المعلومات.
-   **مفسّر الكود**: يسمح للنماذج بكتابة وتشغيل كود Python في بيئة معزولة (sandboxed environment).

> **الاستخدام مع عقدة AI Agent**
>
> يتم دعم الأدوات المدمجة فقط عند استخدام عقدة نموذج دردشة OpenAI (OpenAI Chat Model) بالاشتراك مع عقدة AI Agent. لا تتوفر الأدوات المدمجة عند استخدام عقدة نموذج دردشة OpenAI بالاشتراك مع عقدة سلسلة LLM أساسية (Basic LLM Chain)، على سبيل المثال.

## خيارات العقدة

استخدم هذه الخيارات لتحسين سلوك العقدة بشكل أكبر. تتوفر الخيارات التالية سواء كنت تستخدم Responses API لإنشاء مخرجات النموذج أم لا.

### عقوبة التكرار

استخدم هذا الخيار للتحكم في احتمالية تكرار النموذج لنفسه. القيم الأعلى تقلل من احتمالية تكرار النموذج لنفسه.

### الحد الأقصى لعدد الرموز (Tokens)

أدخل الحد الأقصى لعدد الرموز المستخدمة، والذي يحدد طول الإكمال.

### عقوبة الوجود

استخدم هذا الخيار للتحكم في احتمالية تحدث النموذج عن مواضيع جديدة. القيم الأعلى تزيد من احتمالية تحدث النموذج عن مواضيع جديدة.

### درجة حرارة أخذ العينات

استخدم هذا الخيار للتحكم في عشوائية عملية أخذ العينات. درجة الحرارة الأعلى تخلق أخذ عينات أكثر تنوعًا، ولكنها تزيد من خطر الهلوسات.

### مهلة الانتهاء

أدخل الحد الأقصى لوقت الطلب بالمللي ثانية.

### الحد الأقصى لإعادة المحاولات

أدخل الحد الأقصى لعدد مرات إعادة محاولة الطلب.

### Top P

استخدم هذا الخيار لتعيين الاحتمالية التي يجب أن يستخدمها الإكمال. استخدم قيمة أقل لتجاهل الخيارات الأقل احتمالاً.

## خيارات العقدة الإضافية (Responses API فقط)
تتوفر الخيارات الإضافية التالية عند التبديل إلى **استخدام Responses API**.

### Conversation ID
المحادثة التي تنتمي إليها هذه الاستجابة. تُضاف عناصر المدخلات وعناصر المخرجات من هذه الاستجابة تلقائيًا إلى هذه المحادثة بعد اكتمالها.

### Prompt Cache Key
استخدم هذا المفتاح لتخزين الطلبات المتشابهة مؤقتًا لتحسين معدلات نجاح التخزين المؤقت.

### Safety Identifier
طبّق مُعرّفًا لتتبع المستخدمين الذين قد ينتهكون سياسات الاستخدام.

### Service Tier
اختر مستوى الخدمة الذي يناسب احتياجاتك: Auto، Flex، Default، أو Priority.

### Metadata
مجموعة من أزواج المفتاح-القيمة لتخزين المعلومات المهيكلة. يمكنك إرفاق ما يصل إلى 16 زوجًا بكائن، وهو أمر مفيد لإضافة بيانات مخصصة يمكن استخدامها للبحث بواسطة API أو في لوحة التحكم.

### Top Logprobs
حدد عددًا صحيحًا بين 0 و 20 يحدد عدد الرموز الأكثر احتمالاً التي ستُعاد في كل موضع رمز، كل منها مع احتمال لوغاريتمي مرتبط به.

### Output Format
اختر تنسيق الاستجابة: Text، JSON Schema، أو JSON Object. يُوصى باستخدام JSON Schema، إذا كنت ترغب في تلقي البيانات بتنسيق JSON.

### Prompt
قم بتكوين الموجه (prompt) المعبأ بمعرّف فريد، وإصداره، ومتغيرات قابلة للاستبدال. تُكوّن الموجهات (prompts) عبر لوحة تحكم OpenAI.
