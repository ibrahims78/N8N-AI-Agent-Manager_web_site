# عقدة ClickUp (كليك أب)

استخدم عقدة ClickUp لأتمتة العمل في ClickUp، ودمج ClickUp مع تطبيقات أخرى. تدعم n8n بشكل مدمج مجموعة واسعة من ميزات ClickUp، بما في ذلك إنشاء المجلدات وقوائم المراجعة والوسوم والتعليقات والأهداف، واستردادها وحذفها وتحديثها.

في هذه الصفحة، ستجد قائمة بالعمليات التي تدعمها عقدة ClickUp وروابط لموارد إضافية.

> **بيانات الاعتماد**
>
> ارجع إلى [بيانات اعتماد ClickUp](/integrations/builtin/credentials/clickup.md) للحصول على إرشادات حول إعداد المصادقة.

> **يمكن استخدام هذه العقدة كأداة ذكاء اصطناعي**
>
> يمكن استخدام هذه العقدة لتعزيز قدرات وكيل الذكاء الاصطناعي. عند استخدامها بهذه الطريقة، يمكن تعيين العديد من المعاملات تلقائيًا، أو بمعلومات موجهة بواسطة الذكاء الاصطناعي - اكتشف المزيد في [توثيق معاملات أداة الذكاء الاصطناعي](/advanced-ai/examples/using-the-fromai-function.md).

## تفاصيل العملية

### الحصول على مهمة

عند استخدام عملية **الحصول على مهمة**، يمكنك تمكين الخيارات التالية (اختياريًا):

-   **تضمين المهام الفرعية (Include Subtasks)**: عند التمكين، يتم جلب وتضمين المهام الفرعية للمهمة المحددة أيضًا.
-   **تضمين وصف Markdown (Include Markdown Description)**: عند التمكين، يتم تضمين حقل `markdown_description` في الاستجابة، مما يحافظ على الروابط والتنسيق في وصف المهمة. هذا مفيد إذا كانت أوصاف مهامك تحتوي على روابط أو تنسيق غني.

## ماذا تفعل إذا كانت عمليتك غير مدعومة

إذا كانت هذه العقدة لا تدعم العملية التي ترغب في تنفيذها، يمكنك استخدام [عقدة طلب HTTP (HTTP Request node)](/integrations/builtin/core-nodes/n8n-nodes-base.httprequest/index.md) لاستدعاء واجهة برمجة تطبيقات الخدمة.

يمكنك استخدام بيانات الاعتماد التي أنشأتها لهذه الخدمة في عقدة HTTP Request:

1.  في عقدة HTTP Request، حدد **المصادقة (Authentication)** > **نوع بيانات الاعتماد المُعرّف مسبقًا (Predefined Credential Type)**.
2.  حدد الخدمة التي ترغب في الاتصال بها.
3.  حدد بيانات الاعتماد الخاصة بك.

ارجع إلى [عمليات API المخصصة](/integrations/custom-operations.md) لمزيد من المعلومات.
