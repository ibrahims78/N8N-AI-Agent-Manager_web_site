# عقدة Rundeck

استخدم عقدة Rundeck (Rundeck node) لأتمتة سير العمل في Rundeck، ودمج Rundeck مع التطبيقات الأخرى. يدعم n8n بشكل مدمج تنفيذ الوظائف والحصول على بيانات التعريف.

ستجد في هذه الصفحة قائمة بالعمليات التي تدعمها عقدة Rundeck وروابط لموارد إضافية.

> **بيانات الاعتماد**
>
> ارجع إلى [بيانات اعتماد Rundeck](/integrations/builtin/credentials/rundeck.md) للحصول على إرشادات حول إعداد المصادقة.

## العمليات

-   **الوظيفة (Job)**
    -   تنفيذ وظيفة
    -   الحصول على بيانات تعريف وظيفة

## ماذا تفعل إذا كانت عمليتك غير مدعومة

إذا كانت هذه العقدة لا تدعم العملية التي ترغب في القيام بها، يمكنك استخدام [عقدة HTTP Request](/integrations/builtin/core-nodes/n8n-nodes-base.httprequest/index.md) لاستدعاء API الخاص بالخدمة.

يمكنك استخدام بيانات الاعتماد التي أنشأتها لهذه الخدمة في عقدة HTTP Request:

1.  في عقدة HTTP Request، حدد **Authentication** > **Predefined Credential Type**.
2.  حدد الخدمة التي ترغب في الاتصال بها.
3.  حدد بيانات الاعتماد الخاصة بك.

ارجع إلى [عمليات API المخصصة](/integrations/custom-operations.md) لمزيد من المعلومات.

## العثور على معرّف الوظيفة (job ID)

1.  ادخل إلى لوحة تحكم Rundeck الخاصة بك.
2.  افتح المشروع الذي يحتوي على الوظيفة التي ترغب في استخدامها مع n8n.
3.  في الشريط الجانبي، حدد **JOBS** (الوظائف).
4.  ضمن **All Jobs** (جميع الوظائف)، حدد اسم الوظيفة التي ترغب في استخدامها مع n8n.
5.  في الزاوية العلوية اليسرى، أسفل اسم الوظيفة، انسخ السلسلة النصية المعروضة بخط أصغر أسفل اسم الوظيفة. هذا هو معرّف الوظيفة (job ID) الخاص بك.
6.  الصق معرّف الوظيفة هذا في حقل **Job Id** في n8n.
