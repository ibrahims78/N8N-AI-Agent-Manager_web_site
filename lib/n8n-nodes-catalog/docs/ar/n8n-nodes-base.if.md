# If

استخدم عقدة If لتقسيم سير العمل بشكل شرطي بناءً على عمليات المقارنة.

## إضافة الشروط

أنشئ **شروط** مقارنة لعقدة If الخاصة بك.

- استخدم القائمة المنسدلة لنوع البيانات لتحديد نوع البيانات ونوع عملية المقارنة لشرطك. على سبيل المثال، لتصفية التواريخ التي تلي تاريخًا معينًا، حدد **Date & Time > is after**.
- تتغير الحقول والقيم التي يتم إدخالها في الشرط بناءً على نوع البيانات والمقارنة التي تحددها. ارجع إلى [مقارنات أنواع البيانات المتاحة](#available-data-type-comparisons) للحصول على قائمة كاملة بجميع المقارنات حسب نوع البيانات.

حدد **Add condition** لإنشاء المزيد من الشروط.

### دمج الشروط

يمكنك اختيار الاحتفاظ بالبيانات:

* عندما تستوفي جميع الشروط: أنشئ شرطين أو أكثر وحدد **AND** في القائمة المنسدلة بينهما.
* عندما تستوفي أيًا من الشروط: أنشئ شرطين أو أكثر وحدد **OR** في القائمة المنسدلة بينهما.

## تنفيذ الفروع باستخدام عقدتي If و Merge

> **0.236.0 والإصدارات الأقدم**
>
> أزالت n8n سلوك التنفيذ هذا في الإصدار 1.0. ينطبق هذا القسم على سير العمل التي تستخدم ترتيب تنفيذ سير العمل **v0 (القديم)**. افتراضيًا، يشمل هذا جميع سير العمل التي تم إنشاؤها قبل الإصدار 1.0. يمكنك تغيير ترتيب التنفيذ في [إعدادات سير العمل](/workflows/settings.md) الخاصة بك.
إذا أضفت عقدة Merge إلى سير عمل يحتوي على عقدة If، فقد يؤدي ذلك إلى تنفيذ كلا تدفقي بيانات المخرجات لعقدة If.

يقوم تدفق بيانات واحد بتشغيل عقدة Merge، والتي بدورها تقوم بتنفيذ تدفق البيانات الآخر.

على سبيل المثال، في لقطة الشاشة أدناه يوجد سير عمل يحتوي على عقدة Edit Fields وعقدة If وعقدة Merge. السلوك القياسي لعقدة If هو تنفيذ تدفق بيانات واحد (في لقطة الشاشة، هذا هو المخرجات **true**). ومع ذلك، بسبب عقدة Merge، يتم تنفيذ كلا تدفقي البيانات، على الرغم من أن عقدة If لا ترسل أي بيانات عبر تدفق البيانات **false**.
