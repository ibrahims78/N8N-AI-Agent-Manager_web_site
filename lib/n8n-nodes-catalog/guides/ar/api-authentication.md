# المصادقة على API

يستخدم n8n مفاتيح API للمصادقة على استدعاءات API.

/// info | التوافر على الميزة
لا يتاح API الخاص بـ n8n أثناء الفترة التجريبية المجانية. يرجى الترقية للوصول إلى هذه الميزة.
///

## مجالات الـ API

يمكن لمستخدمي النسخ المؤسسية من n8n تقييد الموارد والإجراءات التي يمكن لمفتاح API الوصول إليها باستخدام المجالات. تسمح مجالات مفتاح API بتحديد مستوى الوصول الدقيق الذي يحتاجه المفتاح لغرضه المقصود.

المفاتيح غير المؤسسية لديها وصول كامل إلى جميع موارد الحساب وقدراته.

## إنشاء مفتاح API

1. تسجيل الدخول إلى n8n.
1. اذهب إلى **الإعدادات** > **n8n API**.
1. حدد **إنشاء مفتاح API**.
1. اختر **Label** وحدد زمن **Expiration** للمفتاح.
1. إذا كنت على خطة مؤسسية، اختر **Scopes** لإعطاء المفتاح.
1. انسخ **My API Key** واستخدم هذا المفتاح للمصادقة على استدعاءاتك.

## استدعاء API باستخدام مفتاحك

أرسل مفتاح API في استدعاء الـ API كـ رأس يحمل الاسم `X-N8N-API-KEY`.

على سبيل المثال، إذا رغبت في الحصول على جميع سير العمل النشط. ستبدو طلبات curl كما يلي:

```shell
# For a self-hosted n8n instance
curl -X 'GET' \
  '<N8N_HOST>:<N8N_PORT>/<N8N_PATH>/api/v<version-number>/workflows?active=true' \
  -H 'accept: application/json' \
  -H 'X-N8N-API-KEY: <your-api-key>'

# For n8n Cloud
curl -X 'GET' \
  '<your-cloud-instance>/api/v<version-number>/workflows?active=true' \
  -H 'accept: application/json' \
  -H 'X-N8N-API-KEY: <your-api-key>'
```

## حذف مفتاح API

1. تسجيل الدخول إلى n8n.
2. اذهب إلى **الإعدادات** > **n8n API**.
3. حدد **حذف** بجوار المفتاح الذي تريد حذفه.
4. أكد الحذف باختيار **احذف إلى الأبد**.
